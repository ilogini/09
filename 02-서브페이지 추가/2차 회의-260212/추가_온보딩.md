# 온보딩/로그인 스크린 - 멍이랑 (withbowwow)

> 앱 최초 실행 시 사용자를 환영하고, 핵심 가치를 전달하며, 계정 생성과 반려동물 등록까지 안내하는 첫인상 스크린.
> Duolingo, Nike Run Club, Pikmin Bloom의 온보딩 베스트 프랙티스를 반려동물 앱에 최적화.

---

## 1. 스크린 개요

### 1.1 목적

온보딩 스크린은 앱을 처음 실행한 사용자에게 멍이랑의 핵심 가치(산책 트래킹, 뱃지 수집, 소셜 경쟁)를 3장의 슬라이드로 직관적으로 전달하고, 소셜 로그인과 반려동물 등록, 필수 권한 설정까지 최소한의 단계로 완료시키는 첫 경험 흐름이다. 온보딩은 앱의 "첫인상"으로, 이 단계에서의 이탈률이 전체 앱 성패를 좌우한다. 목표는 설치 후 5분 이내에 사용자가 첫 산책을 시작할 수 있는 상태로 만드는 것이다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 빠른 이해 | "이 앱이 뭘 하는 앱인지 빨리 알고 싶다" | ✅ 필수 |
| 간편 가입 | "복잡한 회원가입 없이 바로 시작하고 싶다" | ✅ 필수 |
| 반려동물 등록 | "내 반려동물을 등록해서 맞춤 경험을 받고 싶다" | ✅ 필수 |
| 권한 이해 | "왜 위치/알림 권한이 필요한지 알고 싶다" | ✅ 필수 |

### 1.3 진입 경로

```
경로 1: 앱 최초 설치 후 실행 (신규 사용자)
경로 2: 로그아웃 후 재실행 (재로그인)
경로 3: 앱 데이터 삭제 후 실행 (초기화)
```

## 2. 화면 구성

### 스텝 1: 앱 소개 슬라이드 (3장 스와이프)

#### 슬라이드 1: 산책 트래킹 소개

##### 콘텐츠 구성

- **배경**: 라이트 민트 그라데이션 (#D1FAE5 → #FFFFFF)
- **일러스트레이션**: 반려동물과 산책하는 사람 + GPS 경로 라인 (중앙, 240px)
- **헤드라인**: "반려동물과의 산책을 기록해요" (24pt Bold, Dark)
- **설명**: "GPS로 산책 경로, 거리, 시간을 자동 기록하고\n매일의 산책을 소중한 추억으로 남기세요" (15pt Regular, Gray)
- **페이지 인디케이터**: 3개 도트 (1번 활성: Primary, 2-3번: Light Gray)
- **건너뛰기**: 우상단 "건너뛰기" 텍스트 버튼 (14pt, Gray)

##### 이 콘텐츠가 있는 이유

> 첫 번째 슬라이드에서 앱의 가장 핵심적인 기능(산책 트래킹)을 보여주는 것은 Duolingo의 온보딩에서 "언어를 배워보세요"라는 핵심 가치를 첫 화면에 제시하는 것과 동일한 전략이다.
>
> - **초두 효과(Primacy Effect)**: 심리학에서 처음 접한 정보가 가장 강하게 기억에 남는 현상이다. 앱의 핵심 기능인 "산책 기록"을 첫 슬라이드에 배치하면, 사용자가 "이 앱은 산책 기록 앱"이라는 핵심 인식을 형성한다.
> - **직관적 일러스트레이션**: 텍스트보다 이미지가 인지 처리 속도가 60,000배 빠르다는 연구 결과에 따라, 반려동물+산책+GPS라는 핵심 개념을 하나의 일러스트로 즉시 전달한다.
> - **"건너뛰기" 버튼의 존재 이유**: 재설치 사용자나 앱에 대해 이미 알고 있는 사용자를 위해 온보딩을 건너뛸 수 있는 옵션을 제공한다. 강제 온보딩은 이탈률을 높이므로, 선택적 참여를 보장한다. Nielsen Norman Group의 연구에 따르면 건너뛰기 옵션이 있는 온보딩의 완주율이 오히려 더 높다.

#### 슬라이드 2: 뱃지 수집 소개

##### 콘텐츠 구성

- **배경**: 라이트 골드 그라데이션 (#FEF3C7 → #FFFFFF)
- **일러스트레이션**: 뱃지 컬렉션 그리드 + 글로우 효과 뱃지들 (Apple Fitness 스타일, 중앙, 240px)
  - 획득 뱃지: 풀컬러 + 빛남
  - 미획득 뱃지: 회색 실루엣
- **헤드라인**: "걸을수록 뱃지가 모여요" (24pt Bold, Dark)
- **설명**: "거리, 연속일수, 탐험, 시즌별 뱃지를 수집하고\nApple Fitness처럼 나만의 업적 컬렉션을 완성하세요" (15pt Regular, Gray)
- **강조 요소**: 뱃지 아이콘 3개가 순서대로 "팝" 애니메이션으로 등장 (스케일 0→1 + 바운스)
- **페이지 인디케이터**: 2번 활성

##### 이 콘텐츠가 있는 이유

> 두 번째 슬라이드에서 뱃지 시스템을 소개하는 것은, 기존 "챌린지"에서 "통합 뱃지"로 전환된 핵심 차별점을 강조하기 위함이다. Apple Fitness의 온보딩에서도 "업적 뱃지"를 주요 소개 항목으로 포함한다.
>
> - **수집 욕구 자극**: Pikmin Bloom에서 도감 시스템을 온보딩에서 보여주면 사용자의 수집 본능이 즉시 활성화되는 것처럼, 뱃지 그리드의 "채워진 것 + 빈 것"을 동시에 보여주면 "다 채우고 싶다"는 욕구가 발생한다. 이는 제이가르닉 효과와 수집 심리가 결합된 강력한 동기부여 요소이다.
> - **차별화 포인트 부각**: "Nike Run Club처럼 기록 + Apple Fitness처럼 뱃지 수집"이라는 멍이랑만의 조합을 온보딩에서 명확히 전달하면, 사용자가 다른 산책 앱과의 차이를 즉시 인식한다.
> - **뱃지 팝 애니메이션의 목적**: 정적 이미지보다 애니메이션이 주의를 끄는 효과가 3배 높다. 뱃지가 하나씩 "팝"하며 나타나는 것은 "산책할 때마다 이런 보상이 생긴다"는 기대감을 시각적으로 표현한다.

#### 슬라이드 3: 소셜 경쟁 소개

##### 콘텐츠 구성

- **배경**: 라이트 블루 그라데이션 (#DBEAFE → #FFFFFF)
- **일러스트레이션**: 랭킹 리더보드 + 사용자 아바타들 (중앙, 240px)
  - 1위, 2위, 3위 포디움 스타일
  - 반려동물 프로필 아이콘들
- **헤드라인**: "동네 산책러와 경쟁해요" (24pt Bold, Dark)
- **설명**: "주간 랭킹에서 이웃과 건강하게 경쟁하고\n산책 친구를 만나 함께 걸어보세요" (15pt Regular, Gray)
- **페이지 인디케이터**: 3번 활성
- **CTA 버튼**: "시작하기" (전체 너비, Primary 배경, White 텍스트, 18pt Bold, 높이 56px)
  - "건너뛰기" 버튼 대신 "시작하기" 버튼으로 전환

##### 이 콘텐츠가 있는 이유

> 마지막 슬라이드에서 소셜 경쟁 요소를 보여주고 CTA 버튼으로 마무리하는 것은 Nike Run Club의 온보딩 패턴과 동일하다.
>
> - **최신 효과(Recency Effect)**: 마지막에 본 정보도 강하게 기억에 남는다. 소셜 경쟁 요소를 마지막에 배치하면, "이 앱에서 다른 사람과 경쟁할 수 있다"는 인식이 로그인 직후의 행동에 영향을 미친다. 가입 직후 랭킹 탭을 먼저 확인하는 사용자가 증가한다.
> - **사회적 동기의 강력함**: 자기결정이론(Self-Determination Theory)에서 "관계성(Relatedness)"은 내재적 동기의 핵심 요소이다. "혼자 산책하는 것"에서 "다른 사람과 함께/경쟁하며 산책하는 것"으로 프레이밍을 전환하면 앱 사용 동기가 극대화된다.
> - **CTA 전환 타이밍**: 3장의 슬라이드를 모두 본 사용자는 앱의 핵심 가치를 충분히 이해한 상태이다. 이 시점에 "시작하기" 버튼을 제시하면 전환 의향이 가장 높다. 로그인/가입의 마찰을 최소화하기 위해 바로 소셜 로그인 화면으로 연결한다.

### 스텝 2: 소셜 로그인

#### 콘텐츠 구성

- **상단 영역**:
  - 앱 로고: 멍이랑 로고 (80px, 중앙)
  - 환영 메시지: "멍이랑에 오신 것을 환영합니다!" (20pt Bold, 중앙)
  - 서브 메시지: "간편하게 로그인하고 산책을 시작하세요" (14pt, Gray, 중앙)

- **소셜 로그인 버튼** (수직 배치, 간격 12px):

| 버튼 | 색상 | 아이콘 | 텍스트 |
|------|------|--------|--------|
| 카카오 로그인 | #FEE500 (노랑) | 카카오 말풍선 | "카카오로 시작하기" |
| 네이버 로그인 | #03C75A (초록) | 네이버 N | "네이버로 시작하기" |
| Apple 로그인 | #000000 (검정) | Apple 로고 | "Apple로 시작하기" |

- **버튼 스타일**: 전체 너비, 높이 52px, 라운드 코너 12px, 좌측 아이콘 + 중앙 텍스트
- **Apple 로그인**: iOS에서만 표시 (Apple 로그인 가이드라인 준수)
- **약관 동의**: 하단에 "시작하기를 누르면 [이용약관]과 [개인정보처리방침]에 동의합니다" (12pt, Gray, 링크 밑줄)

#### 이 콘텐츠가 있는 이유

> 소셜 로그인은 현대 앱의 표준 인증 방식이며, 한국 시장에서 카카오 로그인의 사용률이 가장 높다. 카카오 로그인을 최상단에 배치하는 것은 토스, 당근마켓 등 국내 주요 앱의 패턴을 따른다.
>
> - **가입 마찰 최소화**: 이메일+비밀번호 기반 회원가입은 평균 이탈률이 40-60%에 달한다. 소셜 로그인은 탭 1번으로 가입이 완료되므로 이탈률을 10% 이하로 낮출 수 있다. Supabase Auth의 소셜 로그인 연동을 통해 구현한다.
> - **카카오 최상단 배치**: 한국 시장에서 카카오 로그인 사용률은 약 70%에 달한다. 최상단 배치는 대부분의 사용자가 스크롤 없이 원하는 로그인 방법을 바로 찾을 수 있게 한다 (Fitts의 법칙).
> - **Apple 로그인 필수**: App Store 가이드라인 4.8에 따라 소셜 로그인을 제공하는 앱은 반드시 Apple 로그인도 함께 제공해야 한다. 미준수 시 앱 심사 거절 사유가 된다.
> - **약관 동의 통합**: 별도의 약관 동의 화면을 두는 대신, 로그인 버튼 하단에 "시작하기를 누르면 동의"로 통합하여 단계 수를 줄인다. 유럽(GDPR) 기준과 달리 한국 개인정보보호법에서는 이 패턴이 허용된다.

### 스텝 3: 반려동물 등록 (최초 1회)

#### 콘텐츠 구성

- **헤더**: "반려동물을 등록해주세요" (20pt Bold) + "나중에" 텍스트 버튼 (우상단)
- **프로그레스 바**: 상단에 4단계 중 현재 위치 표시 (슬라이드 → 로그인 → 등록 → 권한)
- **등록 폼 필드**:

| 필드 | 유형 | 필수 | 설명 |
|------|------|------|------|
| 프로필 사진 | 이미지 피커 (원형 120px) | 선택 | 카메라/갤러리 선택, 기본 아바타 제공 |
| 이름 | 텍스트 입력 | ✅ 필수 | "반려동물의 이름을 입력하세요" (최대 10자) |
| 종 | 드롭다운 선택 | ✅ 필수 | 강아지/고양이 → 세부 품종 선택 (자동완성 검색) |
| 나이 | 숫자 입력 + 단위 | 선택 | "나이를 입력하세요" (년/개월 선택) |
| 체중 | 숫자 입력 | 선택 | "체중을 입력하세요" (kg 단위, 소수점 1자리) |

- **폼 인터랙션**:
  - 사진: 탭 시 ActionSheet (카메라 촬영 / 앨범에서 선택 / 기본 아바타 사용)
  - 종: 탭 시 검색 가능한 바텀시트 (인기 품종 상단 표시)
  - 키보드: 다음 필드로 자동 포커스 이동
- **완료 버튼**: "등록하기" (Primary, 전체 너비, 56px)
  - 필수 항목 미입력 시 비활성화 (Gray)
- **"나중에" 선택 시**: 기본 프로필로 임시 생성, 마이페이지에서 추후 편집 가능

#### 이 콘텐츠가 있는 이유

> 반려동물 등록은 Pikmin Bloom에서 첫 Pikmin을 심는 것과 유사한 "초기 투자(Initial Investment)" 행위이다. 사용자가 반려동물 정보를 입력하는 시간과 노력이 앱에 대한 몰입도를 높인다.
>
> - **매몰 비용 효과(Sunk Cost Effect)**: 반려동물 이름, 사진, 종을 등록하는 데 투자한 시간과 노력은 "이미 투자했으니 계속 써야지"라는 심리적 매몰 비용을 형성한다. Nir Eyal의 "Hooked" 모델에서 "Investment" 단계에 해당하며, 앱 이탈 가능성을 낮추는 핵심 요소이다.
> - **개인화의 시작**: 반려동물 이름이 등록되면 앱 전체에서 "초코와 함께 산책해요", "초코의 이번 주 기록" 등 개인화된 메시지가 가능해진다. 개인화된 경험은 비개인화 경험보다 리텐션이 44% 높다는 연구 결과가 있다.
> - **필수/선택 구분 전략**: 이름과 종만 필수로 하고, 나이/체중은 선택으로 두어 등록 완료까지의 마찰을 최소화한다. "완벽한 프로필보다 빠른 시작"을 우선하는 전략이다. 추가 정보는 나중에 마이페이지에서 입력하도록 유도한다.
> - **"나중에" 옵션의 중요성**: 반려동물 정보를 바로 입력하고 싶지 않은 사용자(예: 지인 추천으로 탐색 중인 사용자)를 위해 건너뛰기를 제공한다. 강제 등록은 이 시점에서의 이탈률을 높인다.

### 스텝 4: 위치 권한 요청

#### 콘텐츠 구성

- **프리 퍼미션 화면** (OS 다이얼로그 전에 앱 자체 화면):
  - 일러스트레이션: 지도 위 GPS 경로 + 반려동물 아이콘 (중앙, 200px)
  - 헤드라인: "산책 경로를 기록하려면\n위치 권한이 필요해요" (20pt Bold, 중앙)
  - 설명: "GPS로 산책 경로를 기록하고, 주변 공원과 동물병원을 찾기 위해 위치 정보를 사용합니다." (14pt, Gray, 중앙)
  - 권한 상세:
    - "앱 사용 중 위치 접근 (필수)"
    - "백그라운드 위치 접근 (산책 중 화면 꺼도 기록 유지)"
  - CTA 버튼: "위치 권한 허용하기" (Primary, 전체 너비, 56px)
  - 하단 텍스트: "위치 정보는 산책 기록에만 사용되며, 다른 목적으로 수집하지 않습니다" (12pt, Gray)
- **CTA 탭 시**: iOS/Android 네이티브 권한 다이얼로그 호출
  - iOS: "앱을 사용하는 동안 허용" → 후속으로 "항상 허용" 요청 (2단계)
  - Android: "정확한 위치" + "앱 사용 중에만 허용" → 후속으로 "항상 허용" 요청

#### 이 콘텐츠가 있는 이유

> 프리 퍼미션 화면(Pre-Permission Screen)은 Google의 Material Design 가이드라인에서 권장하는 "이중 다이얼로그" 패턴이다. OS 권한 다이얼로그를 직접 보여주기 전에, 왜 이 권한이 필요한지를 먼저 설명하는 것이다.
>
> - **권한 허용률 극대화**: OS 네이티브 다이얼로그를 직접 보여주면 "허용" 비율이 약 50%이지만, 프리 퍼미션 화면에서 이유를 먼저 설명하면 허용률이 약 85%까지 올라간다는 CleverTap의 연구 결과가 있다. iOS에서 위치 권한을 "허용 안 함"으로 설정하면 이후 변경이 번거로우므로, 첫 요청에서 허용을 받는 것이 매우 중요하다.
> - **백그라운드 위치의 필수성**: 산책 트래킹의 핵심 기능은 "화면을 꺼도 GPS 기록이 유지"되는 백그라운드 트래킹이다. 이 권한을 받지 못하면 앱의 핵심 가치가 크게 훼손되므로, 왜 "항상 허용"이 필요한지를 명확히 설명한다.
> - **개인정보 보호 메시지**: "위치 정보는 산책 기록에만 사용"이라는 투명한 메시지는 사용자의 프라이버시 우려를 해소한다. GDPR과 한국 개인정보보호법의 "목적 명시" 원칙에도 부합한다.

### 스텝 5: 알림 권한 요청

#### 콘텐츠 구성

- **프리 퍼미션 화면**:
  - 일러스트레이션: 알림 벨 + 뱃지 획득 알림 카드 미리보기 (중앙, 200px)
  - 헤드라인: "산책 리마인더와\n뱃지 알림을 받아보세요" (20pt Bold, 중앙)
  - 설명: "매일 산책 시간에 알려드리고, 뱃지 획득 시 축하 알림을 보내드려요." (14pt, Gray, 중앙)
  - 알림 유형 미리보기:
    - "산책 시간이에요! 초코가 기다리고 있어요"
    - "새 뱃지를 획득했어요! 10km 클럽"
    - "동네 랭킹 3위로 올라갔어요!"
  - CTA 버튼: "알림 허용하기" (Primary, 전체 너비, 56px)
  - 건너뛰기: "나중에 설정하기" (텍스트 버튼, Gray)
- **CTA 탭 시**: OS 네이티브 알림 권한 다이얼로그 호출

#### 이 콘텐츠가 있는 이유

> 알림 권한은 위치 권한과 달리 "선택적"이므로, 건너뛰기 옵션을 더 눈에 띄게 배치한다. Duolingo에서도 알림 권한 요청 시 "이런 알림을 받게 됩니다"라는 미리보기를 보여준다.
>
> - **알림의 리텐션 효과**: 알림을 허용한 사용자는 허용하지 않은 사용자 대비 3.5배 높은 리텐션을 보인다(Leanplum 연구). 산책 리마인더는 DAU의 핵심 동력이므로, 알림 권한 허용이 앱 성공에 직접적으로 영향을 미친다.
> - **알림 미리보기의 설득력**: 추상적인 "알림을 받으시겠습니까?" 대신, 실제로 받게 될 알림의 예시를 보여주면 사용자가 알림의 가치를 구체적으로 인지하여 허용률이 높아진다.
> - **뱃지 알림 강조**: "뱃지 획득 축하 알림"은 보상성 알림이므로, 사용자가 "받고 싶은 알림"으로 인식한다. 보상성 알림의 존재를 먼저 알리면, 이후 리마인더 알림(행동 유도 알림)도 함께 수용하게 되는 "Foot-in-the-Door" 효과가 발생한다.

### 스텝 6: 온보딩 완료 + 첫 산책 유도

#### 콘텐츠 구성

- **축하 화면**:
  - 일러스트레이션: 반려동물 + 발자국 경로 + 파티클 효과 (중앙, 200px)
  - 헤드라인: "준비 완료!" (28pt Bold, Primary)
  - 서브 메시지: "초코와의 첫 산책을 시작해볼까요?" (16pt, Gray)
  - 획득 뱃지 미리보기: "첫 산책을 완료하면 '첫 발자국' 뱃지를 획득합니다!" (14pt, Primary)
    - 뱃지 아이콘 미리보기 (잠금 상태, 64px)
- **CTA 버튼**: "첫 산책 시작하기" (Primary, 전체 너비, 56px)
- **건너뛰기**: "나중에 시작할게요" → 홈 화면으로 이동

#### 이 콘텐츠가 있는 이유

> Pikmin Bloom에서 온보딩 직후 "첫 Pikmin을 심어보세요!"라는 즉각적 행동 유도가 있고, Nike Run Club에서도 가입 직후 "첫 러닝을 시작하세요!"라는 CTA가 제시된다.
>
> - **활성화 이벤트(Activation Event) 유도**: 그로스 해킹에서 "Aha Moment"는 사용자가 앱의 핵심 가치를 체험하는 순간이다. 멍이랑의 Aha Moment는 "첫 산책을 완료하고 기록이 저장되는 순간"이다. 온보딩 직후 첫 산책을 유도하면, Aha Moment까지의 시간을 최소화하여 리텐션을 극대화할 수 있다.
> - **첫 뱃지의 미리보기**: "첫 발자국" 뱃지를 미리 보여주되 잠금 상태로 표시하면, 사용자는 "저 뱃지를 얻으려면 산책을 해야 하는구나"라는 명확한 목표가 생긴다. Pikmin Bloom에서 첫 Pikmin이 씨앗 상태로 보이는 것과 동일한 "예고 보상(Preview Reward)" 전략이다.
> - **온보딩 완주 축하**: "준비 완료!"라는 축하 메시지는 5단계의 온보딩을 완주한 사용자에게 소소한 성취감을 제공하여, 긍정적인 감정 상태에서 첫 산책으로 이어지도록 한다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조 (온보딩 전체 흐름)

```
Step 1 (인지): 앱 소개 슬라이드 — "이 앱은 이런 앱이야"
  ├── 슬라이드 1: 산책 트래킹 (핵심 기능)
  ├── 슬라이드 2: 뱃지 수집 (게이미피케이션)
  └── 슬라이드 3: 소셜 경쟁 (커뮤니티)
Step 2 (가입): 소셜 로그인 — "간편하게 시작"
Step 3 (투자): 반려동물 등록 — "나만의 경험 만들기"
Step 4 (권한): 위치 권한 — "핵심 기능 활성화"
Step 5 (권한): 알림 권한 — "리텐션 도구 활성화"
Step 6 (행동): 첫 산책 유도 — "Aha Moment 도달"
```

총 6단계이지만, 실제 사용자 입력은 로그인(1탭) + 반려동물 이름/종(2필드) + 권한 허용(2탭) = 최소 5회 인터랙션으로 완료 가능하다. 목표 소요 시간: 2-3분 이내.

### 3.2 인터랙션 패턴

| 인터랙션 | 동작 | 피드백 |
|---------|------|--------|
| 슬라이드 스와이프 | 좌우 드래그 | 페이지 전환 + 인디케이터 이동 |
| 건너뛰기 탭 | 텍스트 버튼 | 로그인 화면으로 직접 이동 |
| 소셜 로그인 버튼 | 버튼 탭 | 해당 서비스 인증 화면 (WebView 또는 앱 전환) |
| 사진 촬영/선택 | ActionSheet 선택 | 카메라/갤러리 실행 → 크롭 → 미리보기 |
| 품종 검색 | 바텀시트 내 검색 | 실시간 자동완성 필터링 |
| 권한 허용 | CTA 탭 | OS 다이얼로그 → 허용/거부 결과 처리 |
| 첫 산책 시작 | CTA 탭 | 산책 준비 화면으로 즉시 이동 |

### 3.3 접근성 고려사항

- **VoiceOver/TalkBack**: 모든 슬라이드에 `accessibilityLabel` 적용, 스와이프 대신 좌/우 버튼 대안 제공
- **소셜 로그인 버튼**: 각 버튼에 명확한 레이블 ("카카오 계정으로 로그인")
- **폼 필드**: 라벨과 입력 필드의 연결, 오류 메시지 접근성 트리 포함
- **다크 모드**: 온보딩 전체 다크 테마 대응 (그라데이션 색상 조정)
- **가로 모드**: 슬라이드 레이아웃 가로 모드 대응 (일러스트+텍스트 좌우 배치)
- **터치 영역**: 모든 버튼 최소 44x44pt, 입력 필드 최소 높이 48px
