# 홈 화면 (Tab 1) - 멍이랑 (withbowwow)

> 앱을 열었을 때 가장 먼저 마주하는 화면.
> 사용자가 "오늘 산책해야겠다"는 결심을 3초 안에 하도록 설계한다.

---

## 1. 페이지 개요

### 1.1 목적

홈 화면은 멍이랑 앱의 **첫인상이자 행동 트리거**다. 사용자가 앱을 열었을 때 (1) 오늘의 산책 조건을 즉시 파악하고, (2) 현재 진행 상황을 한눈에 확인하며, (3) 산책 시작이라는 핵심 행동으로 자연스럽게 유도되어야 한다. Nike Run Club이 앱을 열면 "Start" 버튼이 화면 중앙에 위치하여 즉각적인 행동을 이끌어내는 것과 동일한 원리를 적용한다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 오늘 산책 가능한지 확인 | 날씨, 미세먼지 등 외부 조건 판단 | 최상위 |
| 이번 주 얼마나 했는지 | 주간 목표 대비 현재 진행률 확인 | 상위 |
| 바로 산책 시작 | 최소 탭으로 산책 트래킹 진입 | 최상위 |
| 동기부여 요소 확인 | 뱃지 진행률, 랭킹, 기록 등 | 중위 |
| 주변 정보 탐색 | 산책 코스, 동물병원, 반려동물 카페 | 중위 |

### 1.3 진입 경로

| 경로 | 빈도 | 컨텍스트 |
|------|------|---------|
| 앱 최초 실행 | 매번 | 하단 탭 바의 기본 선택 탭 |
| 다른 탭에서 홈 탭 탭 | 높음 | 홈으로 돌아오기 |
| 푸시 알림 탭 → 딥링크 | 중간 | 산책 리마인더, 뱃지 달성 알림 |
| 위젯 탭 | 낮음 | iOS/Android 홈 위젯에서 진입 |

## 2. 화면 구성

### 섹션 0: 앱 헤더 (App Header)

#### 콘텐츠 구성

| 요소 | 위치 | 상세 |
|------|------|------|
| 앱 로고 + 텍스트 | 좌측 | "멍이랑" 로고 (발바닥 아이콘 + 텍스트) |
| 알림 벨 아이콘 | 우측 #1 | 읽지 않은 알림 시 빨간 뱃지 표시 |
| 프로필 아이콘 | 우측 #2 | 대표 반려동물 프로필 사진 (원형, 28px) |

#### 이 콘텐츠가 있는 이유

> 앱 헤더는 사용자에게 "지금 어떤 앱에 있는지"를 즉각적으로 인지시키는 브랜드 앵커 역할을 한다. Instagram, Twitter 등 대부분의 소셜 앱이 좌상단에 로고를 배치하는 것과 동일한 패턴이다.
>
> - **알림 벨**: 챌린지 완료, 랭킹 변동, 소셜 활동 등의 알림을 놓치지 않도록 한다. 빨간 뱃지(unread indicator)는 Zeigarnik 효과를 활용하여 사용자의 탭 행동을 유도한다.
> - **프로필 아이콘**: 반려동물 사진을 표시함으로써 "이 앱은 내 반려동물을 위한 것"이라는 정서적 연결을 강화한다. 사람 얼굴이 아닌 반려동물 사진을 사용하는 것은 멍이랑의 프라이버시 정책(견주 개인정보 비공개)과도 일치한다.
> - **벤치마킹**: Nike Run Club은 상단에 미니멀한 브랜드 로고 + 프로필 아바타를 배치한다.

### 섹션 1: 날씨 위젯 (Weather Widget)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 날씨 아이콘 | 맑음/흐림/비/눈 등 상태 아이콘 |
| 현재 기온 | "12°C" 형태, 체감온도 병기 |
| 산책 적합도 메시지 | "산책하기 좋은 날이에요!", "미세먼지 주의!" 등 |
| 미세먼지 등급 | 좋음/보통/나쁨/매우나쁨 (색상 코딩) |
| 강수 확률 | "오후 3시 이후 비 예보" 등 시간대 표기 |
| 산책 추천 시간대 | "오전 10시~12시가 최적이에요" |

#### 데이터 소스

```
기상청 단기예보 API: 기온(T1H), 강수형태(PTY), 강수확률(POP)
에어코리아 API: 미세먼지(PM10), 초미세먼지(PM2.5)
```

#### 산책 적합도 판정 로직

| 조건 | 판정 | 메시지 색상 |
|------|------|-----------|
| 맑음 + 미세먼지 좋음 + 10~25°C | 최적 | 초록 (#10B981) |
| 흐림 또는 미세먼지 보통 | 보통 | 노랑 (#F59E0B) |
| 비/눈 또는 미세먼지 나쁨 또는 0°C 이하/35°C 이상 | 주의 | 빨강 (#EF4444) |

#### 이 콘텐츠가 있는 이유

> 반려동물 산책에서 날씨는 의사결정의 **가장 첫 번째 변수**다. 사용자가 앱을 여는 순간 "오늘 산책 가능한가?"를 즉시 판단할 수 있어야 한다. 날씨 앱을 별도로 열어 확인하는 단계를 제거함으로써 앱 내 체류 시간을 확보하고, 산책 의사결정까지의 마찰(friction)을 최소화한다.
>
> - **사용자 행동 심리학**: Fogg Behavior Model에 따르면 행동 = 동기 x 능력 x 트리거다. 날씨 위젯은 "능력"(산책 가능 여부 즉시 판단)을 높이고, "산책하기 좋은 날이에요!"라는 메시지는 "트리거" 역할을 한다.
> - **비즈니스 목표**: 날씨가 좋을 때 산책을 유도하여 DAU(일간 활성 사용자)와 일평균 산책 건수를 높인다. 날씨가 나쁜 날에도 "실내 놀이 추천" 등으로 앱 이탈을 방지할 수 있다.
> - **벤치마킹**: Apple Weather 앱의 "체감 정보" 개념을 반려동물 산책에 특화. Pikmin Bloom도 날씨 요소를 게임플레이에 반영한다(비 오는 날 특별 Pikmin 등장).
> - **반려동물 특화 요소**: 미세먼지는 소형견에게 특히 치명적이므로, 일반 날씨 앱보다 미세먼지 정보를 더 강조한다.

### 섹션 2: 이번 주 산책 요약 카드 (Weekly Summary Card)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 카드 배경 | 라운드 카드 (border-radius: 16px), 그라데이션 배경 |
| 주간 총 거리 | "12.5 km" (큰 숫자, Bold) |
| 주간 총 횟수 | "3회" |
| 주간 총 시간 | "2시간 15분" |
| 주간 목표 진행률 | 프로그레스 바 + 퍼센트 ("70%") |
| 주간 목표 설정 | "목표: 20km/주" (탭하면 목표 수정 가능) |
| 요일별 미니 그래프 | 월~일 7칸 막대 (산책한 날 컬러, 안 한 날 회색) |

#### 이 콘텐츠가 있는 이유

> 주간 요약 카드는 사용자의 **자기효능감(self-efficacy)**을 자극하는 핵심 요소다. "이번 주에 이미 12.5km를 걸었다"는 성취감이 "오늘도 해야지"라는 연속 동기를 만든다.
>
> - **사용자 행동 심리학**: Goal Gradient Effect(목표 접근 효과)에 따르면 사람들은 목표에 가까워질수록 더 열심히 행동한다. 프로그레스 바가 70%를 보여주면 "나머지 30%도 채우자"는 심리가 작동한다. Endowed Progress Effect도 함께 작용하여 이미 진행된 것이 있을 때 완료 의지가 강해진다.
> - **비즈니스 목표**: 주간 목표 시스템은 사용자의 리텐션(retention)을 직접적으로 높인다. 주 단위 목표가 있으면 최소 주 2-3회 앱 진입이 보장된다. Apple Fitness의 "Move Ring 채우기"가 매일 Apple Watch 사용을 유도하는 것과 동일한 메커니즘이다.
> - **벤치마킹**: Nike Run Club의 "Weekly Summary"는 주간 목표 대비 진행률을 메인 대시보드에 표시한다. Apple Fitness의 Activity Ring은 시각적 완성도(원형 고리 채우기)로 강력한 동기부여를 제공한다.
> - **요일별 미니 그래프**: 어떤 요일에 산책했는지 패턴을 보여줌으로써 "화요일은 빠졌네, 오늘 해야지" 같은 자기 인식을 촉진한다.

### 섹션 3: 산책 시작 버튼 (Main CTA - Start Walk Button)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 버튼 형태 | 대형 원형 버튼 (지름 80px), Primary 컬러 배경 |
| 버튼 텍스트 | "산책 시작" |
| 버튼 아이콘 | 발바닥(paw) 아이콘 |
| 애니메이션 | 미세한 pulse 효과 (주의 끌기) |
| 부가 텍스트 | 버튼 하단에 "초코와 함께 산책하기" (대표 반려동물 이름) |

#### 인터랙션 흐름

```
[산책 시작] 탭 → 산책 준비 화면 (바텀시트 또는 전체화면)
  → 반려동물 선택 (멀티펫인 경우)
  → 날씨 확인
  → [산책 시작!] 탭
  → 산책 트래킹 전체화면 오버레이 진입
```

#### 이 콘텐츠가 있는 이유

> 산책 시작 버튼은 멍이랑 앱의 **존재 이유**다. 앱의 모든 요소는 궁극적으로 이 버튼을 누르게 만들기 위해 존재한다. 화면 스크롤 없이 첫 뷰포트 안에 위치해야 하며, 시각적으로 가장 눈에 띄어야 한다.
>
> - **사용자 행동 심리학**: Fitts' Law에 따르면 큰 타겟은 빠르게 도달할 수 있다. 80px 원형 버튼은 엄지손가락 터치에 최적화된 크기다. 또한 Von Restorff 효과(격리 효과)에 의해 주변과 다른 형태(원형)와 색상(Primary)의 요소가 기억에 남고 주의를 끈다.
> - **비즈니스 목표**: 일평균 산책 건수가 핵심 KPI다. CTA 버튼의 전환율(탭률)이 높을수록 산책 데이터가 축적되고, 이는 광고 노출 기회(산책 완료 화면), 프리미엄 전환(상세 통계 잠금), 소셜 인터랙션(피드 콘텐츠)으로 연결된다.
> - **벤치마킹**: Nike Run Club은 하단 중앙에 대형 "Start" 버튼을 배치한다. Strava도 "Record" 버튼을 하단 탭 바 중앙에 플로팅한다. 둘 다 앱의 핵심 행동을 가장 접근성 높은 위치에 배치하는 전략이다.
> - **부가 텍스트의 역할**: "초코와 함께 산책하기"라는 텍스트는 반려동물의 이름을 언급함으로써 정서적 동기를 추가한다. "운동해야지"보다 "초코와 함께"가 더 강력한 행동 트리거가 된다.

### 섹션 4: 진행 중인 뱃지 미리보기 (In-Progress Badges Preview)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "진행 중인 뱃지" + "전체보기 >" 링크 |
| 카드 레이아웃 | 가로 스크롤 (Horizontal ScrollView) |
| 카드 크기 | 120 x 150px 고정 |
| 카드 내용 | 뱃지 아이콘(반투명) + 이름 + 프로그레스 바 + "3.2km / 5km" |
| 최대 표시 수 | 3~5개 (가로 스크롤) |
| 빈 상태 | "새로운 뱃지에 도전해보세요!" |

#### 뱃지 상태 표현

| 상태 | 시각적 표현 |
|------|-----------|
| 획득(earned) | 풀 컬러 + 글로우 이펙트 + 날짜 표시 |
| 진행중(in_progress) | 반투명 아이콘 + 프로그레스 바 + 남은 수치 |
| 미발견(locked) | 회색 실루엣 + 자물쇠 아이콘 |

#### 표시 우선순위

1. 달성 임박 뱃지 (80% 이상 진행) - 가장 왼쪽
2. 활발히 진행 중인 뱃지 (20~80%)
3. 최근 시작한 뱃지 (20% 미만)
4. 시즌 한정 뱃지 (남은 기간 표시)

#### 이 콘텐츠가 있는 이유

> 기존 기획에서 "챌린지"로 분리되어 있던 시스템을 **통합 뱃지 시스템**으로 리디자인한 핵심 영역이다. 기존의 "진행 중인 챌린지" 가로 스크롤이 "진행 중인 뱃지"로 대체되며, 사용자 경험은 유사하되 개념적 일관성이 크게 향상된다.
>
> - **사용자 행동 심리학**: Zeigarnik 효과에 따르면 미완료된 과제는 완료된 과제보다 기억에 더 오래 남는다. "3.2km / 5km"처럼 미완성 상태를 보여주면 사용자는 이를 완료하고 싶은 충동을 느낀다. 또한 Variable Ratio Reinforcement(변동비율 강화)에 의해 다양한 카테고리의 뱃지가 예측 불가능한 보상을 제공하여 지속적 참여를 유도한다.
> - **비즈니스 목표**: 뱃지 진행률은 DAU 리텐션의 핵심 드라이버다. "내일 1.8km만 더 걸으면 5km 뱃지를 획득한다"는 인식은 다음 날 앱 재방문의 강력한 이유가 된다. 프리미엄 사용자에게 "무제한 뱃지 트래킹"을 제공하여 전환을 유도할 수 있다.
> - **벤치마킹**: Apple Fitness의 업적(Awards) 시스템은 진행 중인 목표를 Activity 앱 메인에 표시한다. "이번 달 200% Move Goal 달성까지 남은 거리"가 매일의 운동을 유도한다. Pikmin Bloom은 걸음 수 기반으로 꽃심기 진행률을 보여줘 일상 산책의 동기를 부여한다.
> - **통합의 이점**: 기존에 "챌린지" 탭을 별도로 열어야 확인할 수 있었던 진행 상황을 홈 화면에서 즉시 확인 가능. 뱃지와 챌린지가 같은 시스템이므로 사용자가 두 개의 다른 개념을 학습할 필요가 없다.

### 섹션 5: 동네 랭킹 미리보기 (Local Ranking Preview)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "동네 랭킹" + "전체보기 >" 링크 + 지역명 뱃지 ("성동구") |
| 랭킹 리스트 | 상위 3명 표시 |
| 각 항목 구성 | 순위(1/2/3) + 반려동물 프로필 사진 + 닉네임 + 주간 거리 |
| 내 순위 하이라이트 | 나의 항목은 Primary 컬러 배경으로 강조 |
| 기간 표시 | "이번 주" (월~일 기준) |

#### 랭킹 항목 레이아웃

```
┌────────────────────────────────┐
│  1   [🐕사진]  초코아빠   45.2km  │
│  2   [🐶사진]  나 ← ★   38.1km  │  ← 내 순위 하이라이트
│  3   [🐩사진]  달이맘    32.7km  │
└────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 랭킹은 멍이랑 앱의 게이미피케이션 핵심인 **소셜 경쟁** 요소를 홈 화면에서 즉시 자극하는 역할을 한다.
>
> - **사용자 행동 심리학**: Social Comparison Theory(사회 비교 이론)에 따르면 사람들은 자신의 능력과 의견을 타인과 비교하려는 본능이 있다. 특히 "나보다 약간 위"에 있는 사람과의 비교(Upward Comparison)가 가장 강력한 동기부여를 제공한다. "2위인데 1위와 7.1km 차이"라면 "이번 주 안에 따라잡을 수 있겠다"는 계산이 자동으로 시작된다.
> - **비즈니스 목표**: 랭킹 경쟁은 주간 리텐션의 핵심이다. "이번 주 1위"를 유지하려면 매일 산책해야 하므로 DAU가 자연스럽게 높아진다. 동네 기반 랭킹은 실제 이웃과의 경쟁 의식을 통해 더 강한 동기를 부여한다.
> - **벤치마킹**: Nike Run Club의 "Leaderboard"는 친구들의 주간 마일리지를 보여주며 경쟁을 유도한다. Strava의 "Segment Leaderboard"는 특정 구간에서의 순위를 표시하여 반복 사용을 유도한다. 멍이랑은 "동네" 단위로 범위를 좁혀 더 밀접한 경쟁감을 제공한다.
> - **반려동물 프로필 표시**: 랭킹에 사람 정보가 아닌 반려동물 프로필이 표시되므로, 경쟁이 공격적이지 않고 "우리 동네 강아지들의 산책 대결"이라는 유쾌한 프레이밍이 가능하다.

### 섹션 6: 내 주변 (Nearby - Map + Places)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "내 주변" + "지도보기 >" 링크 |
| 미니 지도 | 현재 위치 중심 반경 1km 지도 (높이 150px) |
| 장소 카테고리 필터 | 전체 / 산책코스 / 동물병원 / 반려카페 / 공원 |
| 장소 리스트 | 카드형 가로 스크롤 (이미지 + 이름 + 거리 + 별점) |
| 최대 표시 수 | 5개 (가로 스크롤) |

#### 장소 유형별 아이콘

| 유형 | 아이콘 | 데이터 소스 |
|------|--------|-----------|
| 추천 산책코스 | 경로 아이콘 | 사용자 생성 + 큐레이션 |
| 동물병원 | 병원 아이콘 | 공공데이터 API |
| 동물약국 | 약국 아이콘 | 공공데이터 API |
| 반려동물 카페 | 카페 아이콘 | 카카오 로컬 API |
| 반려동물 공원 | 공원 아이콘 | 공공데이터 API |

#### 이 콘텐츠가 있는 이유

> "내 주변" 섹션은 산책의 **목적지**를 제공하여 무목적 산책을 목적 있는 탐험으로 전환한다.
>
> - **사용자 행동 심리학**: Decision Fatigue(결정 피로)를 줄여준다. "어디로 산책하지?"라는 고민을 앱이 대신 해결해 줌으로써 산책 시작까지의 심리적 장벽을 낮춘다. 또한 Curiosity Gap(호기심 갭)을 활용하여 "이 근처에 이런 곳이 있었어?"라는 발견의 즐거움을 제공한다.
> - **비즈니스 목표**: 지도 내 스폰서 핀(B2B 광고)의 배치 영역이다. 동물병원, 반려동물 카페 등이 유료로 상위 노출을 요청할 수 있다. 또한 탐험 뱃지 시스템("3곳 방문", "10곳 방문")과 연동하여 장소 방문을 유도하고, 이는 다시 뱃지 진행률 → 산책 동기 → DAU 상승으로 이어진다.
> - **벤치마킹**: Pikmin Bloom은 지도 위에 수집 가능한 묘종(seedling)을 표시하여 특정 장소로의 이동을 유도한다. Pokemon GO의 "Nearby" 시스템도 주변 탐험을 유도하는 동일한 메커니즘이다.

### 섹션 7: 광고 배너 (Ad Banner)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 배너 형태 | 네이티브 광고 카드 (320 x 100px) |
| 광고 플랫폼 | 카카오 AdFit |
| "광고" 라벨 | 좌상단에 "AD" 또는 "광고" 텍스트 표시 |
| 닫기 버튼 | 우상단 X 버튼 (해당 세션에서만 숨김) |
| 프리미엄 사용자 | 광고 미표시 |

#### 이 콘텐츠가 있는 이유

> 광고는 멍이랑의 핵심 수익 모델(CPM 1,000~3,000원)이다.
>
> - **배치 전략**: 날씨 위젯, 주간 요약, 산책 시작 버튼 등 핵심 콘텐츠와 뱃지/랭킹 등 부가 콘텐츠 사이에 배치하여, 핵심 기능 사용에는 방해를 주지 않으면서 자연스러운 스크롤 흐름에 녹아들도록 한다.
> - **비즈니스 목표**: 무료 사용자 기반의 광고 수익이 초기 서비스 운영 비용을 충당한다. 동시에 광고가 프리미엄 전환의 인센티브 역할을 한다("광고 제거" 혜택).
> - **UX 고려**: 산책 중 화면에는 광고를 넣지 않고 산책 완료 화면에만 넣되, 홈 화면에서는 콘텐츠 사이에 자연스럽게 삽입한다. Instagram, Facebook 피드 광고 패턴 참고.

### 섹션 8: 최근 산책 기록 (Recent Walk History)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "최근 산책" + "전체보기 >" 링크 |
| 리스트 형태 | 수직 리스트, 최근 3건 표시 |
| 각 항목 구성 | 날짜 + 반려동물 이름 + 미니 경로맵 + 거리 + 시간 |
| 빈 상태 | "아직 산책 기록이 없어요. 첫 산책을 시작해보세요!" |
| 탭 액션 | 항목 탭 → 산책 상세 스크린 |

#### 각 기록 카드 레이아웃

```
┌──────────────────────────────────┐
│  [미니맵]  2월 11일 (화) 오후 3시  │
│            초코와 함께             │
│            2.34km · 32분 · 128kcal│
└──────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 최근 산책 기록은 사용자의 **활동 이력**을 상기시켜 지속적인 참여를 유도한다.
>
> - **사용자 행동 심리학**: Sunk Cost Effect(매몰비용 효과)와 Consistency Principle(일관성 원리)이 작용한다. "어제도 산책했으니 오늘도 해야지"라는 심리가 기록을 보면서 강화된다. 또한 Past Performance Effect에 의해 과거 기록이 미래 행동의 기준점(anchor)이 된다.
> - **비즈니스 목표**: 산책 기록의 축적은 앱의 Lock-in Effect(잠금 효과)를 만든다. 3개월치 산책 데이터가 쌓인 사용자는 다른 앱으로 이동하기 어렵다. 또한 미니 경로맵의 시각적 매력이 "이 기록을 SNS에 공유하고 싶다"는 욕구를 자극하여 바이럴 효과를 만든다.
> - **벤치마킹**: Nike Run Club은 최근 달리기 기록을 경로맵과 함께 리스트로 보여준다. Strava도 Activity Feed에서 최근 활동을 시간순으로 표시한다.

### 섹션 9: 프리미엄 배너 (Premium Promotion Banner)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 배너 형태 | 풀 너비 카드, 그라데이션 배경 (Primary → Secondary) |
| 헤드라인 | "멍이랑 프리미엄으로 업그레이드" |
| 서브텍스트 | "광고 제거, 상세 통계, 무제한 뱃지 트래킹" |
| CTA 버튼 | "무료 체험 시작" |
| 닫기 버튼 | X 버튼 (7일간 숨김) |
| 조건 | 무료 사용자에게만 표시 |

#### 이 콘텐츠가 있는 이유

> 프리미엄 배너는 **수익화 퍼널(monetization funnel)**의 첫 번째 터치포인트다.
>
> - **배치 전략**: 홈 화면 하단에 배치하여 핵심 기능 사용을 방해하지 않으면서도, 스크롤을 끝까지 하는 적극적 사용자에게 노출된다. 적극적 사용자일수록 프리미엄 전환 가능성이 높다.
> - **비즈니스 목표**: 프리미엄 구독(월 3,900원)은 장기 수익의 핵심이다. 무료 사용에서 충분한 가치를 경험한 뒤 "더 원하는" 순간에 전환을 제안한다. 3개월 목표 전환율 3%, 6개월 목표 5%.
> - **벤치마킹**: Spotify의 무료 사용자 대상 프리미엄 배너 전략. 기능 사용 중 "이 기능은 프리미엄에서 가능합니다" 인라인 프로모션.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
[1순위 - 즉각 행동]
  └── 날씨 위젯 (산책 가능 여부 즉시 판단)
  └── 산책 시작 버튼 (핵심 CTA)

[2순위 - 동기부여]
  └── 주간 요약 카드 (자기효능감)
  └── 진행 중 뱃지 (미완료 과제 효과)
  └── 동네 랭킹 (사회 비교)

[3순위 - 탐색/부가]
  └── 내 주변 (목적지 제안)
  └── 최근 기록 (활동 이력)
  └── 광고/프리미엄 (수익화)
```

핵심 원칙: **위에서 아래로 갈수록 행동 촉구 → 동기부여 → 정보 제공 순서로 배치**한다. 사용자가 스크롤하지 않고 첫 뷰포트에서 날씨 확인 + 산책 시작이 가능해야 한다.

### 3.2 인터랙션 패턴

| 패턴 | 적용 위치 | 설명 |
|------|----------|------|
| Pull-to-Refresh | 전체 화면 | 아래로 당겨 데이터 새로고침 (날씨, 랭킹 등) |
| Horizontal Scroll | 뱃지 미리보기, 내 주변 장소 | 가로 스크롤로 더 많은 항목 탐색 |
| Tap to Navigate | 전체보기 링크, 기록 항목 | 탭하여 상세 화면으로 이동 |
| Skeleton Loading | 데이터 로딩 중 | API 응답 대기 중 뼈대 UI 표시 |
| Haptic Feedback | 산책 시작 버튼 탭 | 가벼운 진동으로 행동 확인 |
| Lottie Animation | 뱃지 달성 시 | 축하 애니메이션 오버레이 |

### 3.3 접근성 고려사항

| 항목 | 대응 |
|------|------|
| 색상 대비 | 모든 텍스트 WCAG AA 기준 충족 (4.5:1 이상) |
| 터치 영역 | 모든 인터랙티브 요소 최소 44x44px |
| 스크린 리더 | 모든 이미지에 alt 텍스트, 뱃지 진행률 aria-label |
| 다크 모드 | 시스템 설정 연동, 컬러 팔레트 별도 정의 |
| 글꼴 크기 | 시스템 Dynamic Type 지원 (iOS) / Font Scale (Android) |
| 모션 감소 | Reduce Motion 설정 시 애니메이션 비활성화 |
