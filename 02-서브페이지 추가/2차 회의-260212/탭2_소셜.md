# 소셜 화면 (Tab 2) - 멍이랑 (withbowwow)

> 반려동물 중심의 안전한 소셜 경험.
> 사람이 아닌 "반려동물끼리의 교류"라는 프레이밍으로 설계한다.

---

## 1. 페이지 개요

### 1.1 목적

소셜 탭은 멍이랑 앱의 **커뮤니티 허브**다. 혼자 하는 산책을 "함께 하는 산책"으로 확장하여, 사용자 간 유대감을 형성하고 장기 리텐션을 확보한다. 핵심 원칙은 **견주의 개인정보를 철저히 보호하면서도 반려동물 중심의 따뜻한 소셜 경험을 제공하는 것**이다. 모든 소셜 인터랙션은 반려동물 프로필을 매개로 이루어지며, 사람의 성별, 나이, 직업 등 개인정보는 수집하지 않고 노출하지 않는다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 산책 친구 찾기 | 같은 동네에서 비슷한 시간대에 산책하는 반려동물 찾기 | 최상위 |
| 산책 기록 공유 | 오늘의 산책 경로, 사진을 공유하고 반응 받기 | 상위 |
| 함께 산책하기 | 실시간으로 산책 초대를 보내고 함께 걷기 | 중위 |
| 정보 교류 | 산책 코스 추천, 반려동물 관련 팁 공유 | 중위 |
| 소속감 형성 | 동네 산책 커뮤니티에 소속되어 있다는 느낌 | 상위 |

### 1.3 진입 경로

| 경로 | 빈도 | 컨텍스트 |
|------|------|---------|
| 하단 탭 바 "소셜" 탭 | 높음 | 일상적 소셜 피드 확인 |
| 푸시 알림 (팔로우, 좋아요, 초대) | 중간 | 소셜 알림 딥링크 |
| 홈 → 랭킹 미리보기 → 프로필 탭 → 팔로우 | 낮음 | 랭킹에서 발견한 사용자 팔로우 |
| 산책 완료 → 공유 → 소셜 피드 | 중간 | 산책 기록 공유 후 피드 확인 |

## 2. 화면 구성

### 섹션 1: 상단 네비게이션 바 (Top Navigation)

#### 콘텐츠 구성

| 요소 | 위치 | 상세 |
|------|------|------|
| 페이지 타이틀 | 좌측 | "소셜" (Bold, 24pt) |
| 산책 초대 아이콘 | 우측 #1 | 받은 초대가 있을 시 뱃지 표시 |
| DM/채팅 아이콘 | 우측 #2 | 향후 확장 (Phase 2) |

#### 이 콘텐츠가 있는 이유

> 소셜 탭의 상단 네비게이션은 Instagram의 상단 바와 유사한 패턴을 따른다. 좌측 타이틀로 현재 위치를 명확히 하고, 우측 아이콘으로 알림성 기능에 빠르게 접근할 수 있도록 한다.
>
> - **산책 초대 아이콘**: 산책 초대는 소셜 탭의 핵심 기능이므로 항상 접근 가능한 위치에 배치한다. 읽지 않은 초대가 있으면 빨간 뱃지를 표시하여 Zeigarnik 효과를 활용한다.
> - **비즈니스 목표**: 소셜 기능의 활성화는 네트워크 효과(Network Effect)를 만든다. 친구가 많을수록 앱을 떠나기 어려워지고, 이는 LTV(Lifetime Value) 향상으로 직결된다.

### 섹션 2: 동네 산책 친구 추천 (Nearby Walk Friends)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "동네 산책 친구" + 지역명 뱃지 ("성동구") |
| 추천 카드 | 가로 스크롤 카드 리스트 |
| 카드 크기 | 140 x 200px |
| 카드 내용 | 반려동물 사진(원형) + 이름 + 견종 + 산책 시간대 + [팔로우] 버튼 |
| 추천 기준 | 같은 동네 + 비슷한 산책 시간대 + 비슷한 견종/크기 |
| 최대 표시 수 | 5~8개 (가로 스크롤) |
| 새로고침 | 아래로 당겨 새 추천 로드 |

#### 추천 카드 레이아웃

```
┌──────────┐
│  [🐕📷]  │  ← 반려동물 프로필 사진 (원형, 64px)
│   초코    │  ← 반려동물 이름
│  골든리트리버│ ← 견종
│  오후 산책  │  ← 산책 시간대
│ [+ 팔로우] │  ← CTA 버튼
└──────────┘
```

#### 추천 알고리즘 기준

| 기준 | 가중치 | 설명 |
|------|--------|------|
| 지리적 근접성 | 40% | GPS 기반 동일 행정구/동 |
| 산책 시간대 유사성 | 25% | 최근 7일 산책 시작 시간 비교 |
| 견종/크기 유사성 | 20% | 소형/중형/대형 매칭 (유사 크기끼리) |
| 활동량 유사성 | 15% | 주간 산책 거리/횟수 유사 |

#### 이 콘텐츠가 있는 이유

> 동네 산책 친구 추천은 소셜 탭의 **성장 엔진**이다. 사용자가 팔로우할 사람을 직접 찾아야 하면 소셜 기능이 활성화되지 않는다. 앱이 먼저 적합한 산책 친구를 추천함으로써 Cold Start Problem(콜드 스타트 문제)을 해결한다.
>
> - **사용자 행동 심리학**: Propinquity Effect(근접성 효과)에 따르면 물리적으로 가까이 있는 사람들끼리 친밀감이 형성되기 쉽다. 같은 동네, 비슷한 시간대에 산책하는 반려동물을 추천하면 "아, 이 강아지 산책할 때 봤던 것 같은데!"라는 인식이 가능하다. 또한 Homophily(유사성 원리)에 의해 비슷한 견종/크기의 반려동물끼리 매칭하면 실제 산책 시 궁합이 좋다.
> - **비즈니스 목표**: 소셜 연결 수가 늘어날수록 앱의 Network Effect가 강화된다. Metcalfe's Law에 따르면 네트워크의 가치는 사용자 수의 제곱에 비례한다. 팔로우 관계가 형성되면 피드 콘텐츠가 생기고, 이는 앱 재방문 이유가 된다.
> - **벤치마킹**: Instagram의 "Suggested for You"는 알고리즘 기반 팔로우 추천으로 네트워크 성장을 가속한다. Twitter(X)의 "Who to Follow"도 동일한 패턴이다.
> - **프라이버시 설계**: 반려동물 정보만 표시하고 견주 정보는 완전히 비공개. 이는 멍이랑의 핵심 차별화 요소이자 사용자 안전을 위한 설계다. 특히 여성 사용자들이 위치 기반 소셜 서비스에서 느낄 수 있는 안전 우려를 "반려동물 중심" 프레이밍으로 해소한다.

### 섹션 3: 팔로잉/팔로워 리스트 (Following/Followers)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 세그먼트 컨트롤 | [팔로잉] / [팔로워] 탭 전환 |
| 사용자 수 표시 | "팔로잉 23" / "팔로워 47" |
| 리스트 항목 | 반려동물 사진 + 이름 + 견종 + 최근 산책 정보 + [팔로우/팔로잉] 버튼 |
| 검색 바 | 닉네임 또는 반려동물 이름으로 검색 |
| 정렬 옵션 | 최근 산책 순 / 이름 순 / 팔로우 날짜 순 |
| 접기/펼치기 | 기본 3명 표시 + "더보기" 버튼으로 확장 |

#### 리스트 항목 레이아웃

```
┌──────────────────────────────────────┐
│ [🐕📷] 초코 (골든리트리버)   [팔로잉 ✓] │
│         오늘 3.2km 산책 · 1시간 전     │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 팔로잉/팔로워 리스트는 사용자의 **소셜 그래프**를 관리하는 영역이다.
>
> - **사용자 행동 심리학**: Social Proof(사회적 증거)에 의해 팔로워 수가 많은 사용자일수록 앱 내에서 신뢰도와 영향력이 높다. "47명이 나를 팔로우한다"는 사실은 자존감을 높이고 앱에 대한 애착을 강화한다. 또한 Reciprocity Principle(상호성 원리)에 의해 "이 사용자가 나를 팔로우했다"는 알림은 역팔로우 행동을 유도한다.
> - **비즈니스 목표**: 평균 팔로잉 수가 높을수록 피드에 콘텐츠가 풍부해지고, 이는 앱 체류 시간 증가로 이어진다. Facebook의 초기 성장 지표가 "10일 내 7명 이상 친구 추가"였듯, 멍이랑도 "첫 주 내 5명 이상 팔로우"를 핵심 활성화 지표로 삼는다.
> - **"최근 산책 정보" 표시 이유**: 팔로잉 중인 반려동물이 "오늘 3.2km 산책"했다는 정보는 사회 비교를 통한 산책 동기부여 역할을 한다. "다들 산책하고 있네, 나도 나가야겠다."

### 섹션 4: 산책 피드 (Walk Feed Timeline)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 피드 형태 | 수직 무한 스크롤 (Infinite Scroll) |
| 피드 필터 | [전체] / [팔로잉만] / [동네] 세그먼트 |
| 피드 카드 구성 | 프로필 헤더 + 경로 맵 이미지 + 산책 통계 + 사진(선택) + 좋아요/댓글 |
| 좋아요 | 하트 아이콘 탭 (Lottie 애니메이션) |
| 댓글 | 간단한 텍스트 댓글 + 이모지 리액션 |
| 공유 | 카카오톡, 인스타그램 스토리 공유 |

#### 피드 카드 레이아웃

```
┌──────────────────────────────────┐
│ [🐕📷] 초코 · 성동구 · 2시간 전    │  ← 프로필 헤더
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │                              │ │
│ │      [산책 경로 맵 이미지]     │ │  ← 경로 지도 캡처
│ │                              │ │
│ └──────────────────────────────┘ │
├──────────────────────────────────┤
│  📏 3.2km  ⏱️ 45분  🔥 186kcal   │  ← 산책 통계
├──────────────────────────────────┤
│ [📷] [📷] [📷]                    │  ← 산책 중 촬영 사진 (선택)
├──────────────────────────────────┤
│ ❤️ 12   💬 3    📤 공유           │  ← 인터랙션 바
│ "오늘 한강공원 코스 너무 좋았어요"  │  ← 캡션 (선택)
└──────────────────────────────────┘
```

#### 피드 정렬 알고리즘

| 요소 | 가중치 | 설명 |
|------|--------|------|
| 시간 (최신순) | 40% | 최근 게시물 우선 |
| 친밀도 | 30% | 자주 인터랙션하는 사용자 우선 |
| 지역 근접성 | 20% | 같은 동네 우선 |
| 인게이지먼트 | 10% | 좋아요/댓글 많은 콘텐츠 부스트 |

#### 이 콘텐츠가 있는 이유

> 산책 피드는 소셜 탭의 **핵심 콘텐츠 영역**이다. 사용자가 소셜 탭에 머무는 시간의 대부분을 차지한다.
>
> - **사용자 행동 심리학**: FOMO(Fear of Missing Out)가 작동한다. 다른 사용자들의 산책 기록을 스크롤하면서 "나도 오늘 산책해야겠다"는 동기가 자연스럽게 생긴다. 또한 Social Facilitation(사회적 촉진)에 의해 다른 사람들이 산책하는 것을 보면 자신의 산책 동기도 높아진다. Instagram에서 친구들의 운동 사진을 보면 운동 의욕이 높아지는 것과 동일한 메커니즘이다.
> - **비즈니스 목표**: 피드는 앱 체류 시간의 핵심 드라이버다. 체류 시간이 길수록 광고 노출 기회가 증가하고, 피드 사이 네이티브 광고 삽입이 가능하다. 또한 피드 콘텐츠가 풍부해질수록 앱의 가치가 높아져 신규 사용자 유입이 촉진된다.
> - **벤치마킹**: Strava의 Activity Feed는 팔로잉 중인 사용자의 운동 기록을 타임라인으로 보여주며, "Kudos"(좋아요)와 댓글로 인터랙션한다. Nike Run Club의 Activity Feed도 유사한 구조다. 멍이랑은 여기에 "반려동물 중심"이라는 차별화를 더한다.
> - **경로 맵 이미지의 매력**: 산책 경로가 그려진 지도 이미지는 시각적으로 매력적이다. 마치 "오늘 이런 그림을 그렸어요"라는 느낌을 주며, 공유 욕구를 자극한다. Strava의 "Route Art" 문화가 이를 증명한다.

### 섹션 5: 산책 초대 보내기/받기 (Walk Invitation)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 진입점 | 상단 네비게이션 아이콘 + 피드 내 "함께 산책하기" 버튼 |
| 초대 생성 | 시간 설정 + 장소 설정(선택) + 메시지(선택) + 보낼 대상 선택 |
| 받은 초대 카드 | 초대자 반려동물 프로필 + 시간 + 장소 + [수락] / [거절] |
| 초대 상태 | 대기중 / 수락됨 / 거절됨 / 만료됨 |
| 만료 시간 | 산책 예정 시간 30분 전 자동 만료 |
| 푸시 알림 | 초대 수신 시 즉시 알림 |

#### 초대 카드 레이아웃

```
┌──────────────────────────────────┐
│  🐾 산책 초대가 도착했어요!         │
├──────────────────────────────────┤
│  [🐕📷] 초코가 함께 산책하자고      │
│         합니다!                    │
│                                  │
│  📅 오늘 오후 5시                  │
│  📍 한강공원 뚝섬입구              │
│  💬 "오늘 날씨 좋은데 같이 가요~"   │
│                                  │
│  [✓ 수락하기]    [✕ 다음에]        │
└──────────────────────────────────┘
```

#### 함께 산책 진행 흐름

```
초대 수락 → 약속 시간 알림 → 각자 산책 시작
→ 실시간 위치 공유 (선택) → 만남 후 함께 산책
→ 각자 산책 완료 기록 → "함께 산책" 뱃지 표시
```

#### 이 콘텐츠가 있는 이유

> 산책 초대는 "디지털 소셜"을 "오프라인 만남"으로 연결하는 **브릿지 기능**이다.
>
> - **사용자 행동 심리학**: Commitment and Consistency(약속과 일관성)에 의해 산책 약속을 하면 실제 산책 이행률이 크게 높아진다. 혼자 산책 계획은 쉽게 미루지만, 누군가와 약속한 산책은 지키게 된다. 운동 파트너가 있으면 운동 지속률이 40-65% 향상된다는 연구 결과가 이를 뒷받침한다.
> - **비즈니스 목표**: 함께 산책 기능은 앱의 사용 빈도를 높이는 동시에 바이럴 효과를 만든다. 산책 초대를 받기 위해서는 상대방도 앱을 설치해야 하므로, 자연스러운 사용자 획득(organic acquisition) 채널이 된다.
> - **벤치마킹**: Nike Run Club의 "Club Run" 기능은 그룹 달리기를 조직할 수 있다. Strava의 "Group Activities"도 유사하다. 멍이랑은 반려동물의 궁합(크기, 성격)도 고려한 매칭으로 차별화한다.
> - **안전 설계**: 정확한 위치 공유는 산책 시작 후에만 활성화되며, 개인 주소 등은 절대 노출하지 않는다. 만남 장소는 공원, 산책로 등 공공장소로 제한 추천한다.

### 섹션 6: 산책 모임 게시판 (Walk Meetup Board)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "산책 모임" + "모임 만들기" 버튼 |
| 게시글 형태 | 카드형 리스트 (수직 스크롤) |
| 카드 내용 | 모임명 + 장소 + 시간 + 참가자 수/제한 + 반려동물 크기 제한 |
| 카테고리 필터 | 전체 / 소형견 / 중형견 / 대형견 / 견종별 |
| 참가 방식 | [참가 신청] 버튼 → 모임장 승인/자동 승인 |
| 정기 모임 | 매주 반복 설정 가능 (매주 토요일 오전 10시 등) |

#### 모임 카드 레이아웃

```
┌──────────────────────────────────┐
│  🏷️ 성수동 골든리트리버 모임       │
├──────────────────────────────────┤
│  📅 매주 토요일 오전 10시          │
│  📍 서울숲 메타세쿼이아길          │
│  🐕 대형견 환영 · 소형견 가능      │
│  👥 참가자 8/15명                 │
│  [🐕][🐕][🐕]... +5              │  ← 참가자 반려동물 프로필
│                                  │
│  [참가 신청하기]                   │
└──────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 산책 모임 게시판은 1:1 산책 초대를 넘어 **커뮤니티 형성**을 촉진하는 기능이다.
>
> - **사용자 행동 심리학**: Belonging(소속감 욕구)은 Maslow의 욕구 계층에서 3번째 단계다. 정기 산책 모임에 참여하면 "나는 성수동 골든리트리버 모임의 멤버"라는 정체성이 형성되고, 이는 앱 이탈을 방지하는 강력한 Lock-in 요소가 된다. 또한 Community of Practice(실천 공동체) 개념에 의해 공통 관심사(특정 견종, 산책)를 가진 그룹이 지속적으로 만나면서 지식과 경험을 교환한다.
> - **비즈니스 목표**: 모임 기능은 앱의 MAU(Monthly Active Users) 리텐션에 직접적으로 기여한다. "매주 토요일 모임"이 있으면 최소 주 1회 앱 방문이 보장된다. 또한 모임 게시판은 네이티브 광고(반려동물 용품, 펫 카페 등)의 자연스러운 배치 공간이다.
> - **벤치마킹**: Meetup 앱의 그룹/이벤트 구조를 참고. 당근마켓의 "동네생활" 게시판이 지역 커뮤니티 형성에 성공한 사례도 참고한다. 네이버 카페의 "정기 모임" 기능도 벤치마킹 대상이다.
> - **견종별 카테고리의 필요성**: 소형견과 대형견의 산책 페이스, 에너지 레벨이 다르므로 견종/크기별 모임이 실용적이다. 또한 같은 견종 보호자들 사이에서 견종 특화 정보(건강, 훈련 팁)를 교환할 수 있다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
[1순위 - 네트워크 성장]
  └── 동네 산책 친구 추천 (새로운 연결 형성)
  └── 팔로잉/팔로워 리스트 (기존 네트워크 관리)

[2순위 - 콘텐츠 소비]
  └── 산책 피드 (타임라인 스크롤)

[3순위 - 오프라인 연결]
  └── 산책 초대 (1:1 만남)
  └── 산책 모임 게시판 (그룹 활동)
```

핵심 원칙: **위에서 아래로 갈수록 "연결 형성" → "콘텐츠 소비" → "오프라인 행동" 순서**로 배치한다. 먼저 팔로우할 사람을 찾고, 그들의 콘텐츠를 소비하며, 점점 더 깊은 관계(함께 산책, 정기 모임)로 발전하는 퍼널 구조다.

### 3.2 인터랙션 패턴

| 패턴 | 적용 위치 | 설명 |
|------|----------|------|
| Pull-to-Refresh | 피드 전체 | 새 피드 콘텐츠 로드 |
| Horizontal Scroll | 친구 추천 | 가로 스크롤로 추천 카드 탐색 |
| Infinite Scroll | 피드 리스트 | 하단 도달 시 다음 페이지 자동 로드 |
| Double Tap to Like | 피드 카드 | 더블 탭으로 좋아요 (Instagram 패턴) |
| Swipe to Dismiss | 추천 카드 | 관심 없는 추천 제거 |
| Long Press | 피드 카드 | 신고/숨기기 컨텍스트 메뉴 |
| Bottom Sheet | 댓글, 공유 | 바텀시트로 댓글 목록/공유 옵션 표시 |
| Haptic Feedback | 좋아요 탭 | 가벼운 진동으로 인터랙션 확인 |

### 3.3 접근성 고려사항

| 항목 | 대응 |
|------|------|
| 색상 대비 | 좋아요(빨강), 팔로우(Primary) 등 인터랙션 요소 4.5:1 이상 |
| 터치 영역 | 좋아요, 팔로우 버튼 등 최소 44x44px |
| 스크린 리더 | "초코의 산책 기록, 3.2킬로미터, 45분" 등 구조화된 읽기 |
| 이미지 대체 텍스트 | 경로 맵 이미지에 "한강공원 3.2km 산책 경로" 등 alt |
| 키보드 접근성 | 모든 인터랙티브 요소에 포커스 순서 지정 |
| 모션 감소 | 좋아요 애니메이션을 정적 상태 변경으로 대체 |

## 5. 안전 및 신고 시스템

### 5.1 신고 유형

| 유형 | 설명 | 처리 |
|------|------|------|
| 부적절한 콘텐츠 | 반려동물과 무관한 사진/텍스트 | 24시간 내 검토 |
| 스팸/광고 | 무분별한 광고성 게시물 | 자동 필터 + 수동 검토 |
| 괴롭힘/불쾌한 댓글 | 악의적 댓글 | 즉시 숨김 + 검토 |
| 동물 학대 의심 | 학대 의심 콘텐츠 | 긴급 검토 + 관계기관 신고 연계 |
| 개인정보 노출 | 견주 실명, 주소 등 노출 | 즉시 숨김 + 삭제 |

### 5.2 차단 기능

- 특정 사용자 차단 시 상호 피드/추천에서 완전 제외
- 차단 사실은 상대방에게 알리지 않음
- 차단 목록은 설정에서 관리 가능
