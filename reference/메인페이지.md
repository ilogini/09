# 메인페이지 (홈 탭) - 멍이랑 (withbowwow)

## 1. 페이지 개요

### 1.1 목적
앱의 **첫 화면이자 허브 역할**을 하는 메인페이지입니다. 사용자가 앱을 열었을 때 가장 먼저 보는 화면으로, 하단 중앙 플로팅 버튼을 통해 산책을 시작하고, 현재 상태(날씨, 주간 요약, 뱃지 진행률)를 한눈에 파악할 수 있도록 구성합니다.

### 1.2 핵심 사용자 니즈
- "오늘 산책하기 좋은 날인가?" → 날씨 위젯
- "이번 주 얼마나 걸었지?" → 주간 요약 카드
- "뱃지 얼마나 남았지?" → 진행 중인 뱃지
- "주변에 산책 가기 좋은 곳?" → 내 주변 지도/장소

### 1.3 진입 경로
- 앱 실행 시 기본 화면 (Tab 1)
- 다른 탭에서 홈 탭 아이콘 클릭

### 1.4 목업
- `prototype/멍이랑_메인페이지_시안.html`

## 2. 앱 셸 구조 (모든 페이지 공통)

메인페이지는 단순한 "홈 탭"이 아니라, 앱 전체의 **셸(Shell) 구조**를 정의하는 역할도 합니다.

### 2.1 상단 헤더
- **구성**: 앱 로고 (멍이랑) + 알림 벨 아이콘 + 설정 아이콘
- **고정 여부**: `position: sticky` - 스크롤 시에도 항상 상단 고정

#### 이 콘텐츠가 있는 이유
> 헤더는 앱의 아이덴티티를 지속적으로 노출하면서, 알림과 설정이라는 두 가지 핵심 기능에 항상 접근 가능하게 합니다. GPS 뱃지는 제거되었으며, NRC 스타일의 미니멀한 헤더 디자인을 따릅니다. 설정 버튼을 헤더에 배치하여 마이페이지의 설정 섹션을 독립시키고, 어느 탭에서든 설정에 바로 접근할 수 있도록 합니다.

### 2.2 하단 탭 네비게이션
- **구성**: 홈 / 소셜 / 산책(중앙 플로팅) / 지도 / MY (5탭)
- **고정 여부**: `position: fixed` - 화면 하단 고정
- **중앙 산책 버튼**: 다른 탭보다 크고 돌출된 형태, 앱의 핵심 액션 강조. 탭 시 반려동물 선택 모달이 먼저 표시된 후 산책 트래킹 시작.

#### 이 콘텐츠가 있는 이유
> 5탭 하단 네비게이션은 모바일 앱의 표준 패턴으로, 엄지 손가락이 쉽게 닿는 화면 하단에 배치합니다. 중앙의 산책 버튼을 플로팅으로 강조하는 것은 Nike Run Club의 "START" 버튼, Instagram의 "+" 버튼처럼 앱의 핵심 액션을 시각적으로 차별화하는 패턴입니다. 기존 랭킹 탭은 지도 탭으로 변경되었으며, 랭킹 기능은 소셜 탭의 주간 리더보드로 통합되었습니다.

### 2.3 산책 오버레이 (전체화면)
- **구성**: GPS 지도(그레이스케일, 비조작) + 타이머 + 거리/칼로리 + 일시정지/종료/카메라 컨트롤
- **트리거**: 산책 플로팅 버튼 클릭 → 반려동물 선택 모달 → 전체화면 오버레이로 전환
- **상세**: [reference/탭3_산책.md](탭3_산책.md) 참조

## 3. 홈 탭 콘텐츠 구성

### 섹션 1: 날씨 위젯

#### 콘텐츠 구성
- 현재 기온 (대형 숫자)
- 날씨 아이콘
- 산책 적합도 메시지 ("산책하기 좋은 날이에요!")
- 미세먼지/습도 부가 정보

#### 이 콘텐츠가 있는 이유
> 반려동물 산책에서 날씨는 가장 큰 결정 요인입니다. Fogg Behavior Model에 따르면, 행동(산책)은 동기(Motivation) x 능력(Ability) x 촉발(Trigger)의 조합으로 발생합니다. 날씨가 좋다는 정보("산책하기 좋은 날이에요!")는 동기를 즉시 높이는 촉발 역할을 합니다. 기상청 단기예보 API를 연동하여 실시간 데이터를 제공하며, 미세먼지 정보는 반려동물의 호흡기 건강과 직결되므로 함께 표시합니다.

### 섹션 2: 이번 주 산책 요약 카드

#### 콘텐츠 구성
- 총 거리 (km), 산책 횟수, 총 시간
- 기간 표시 (날짜 범위)

#### 이 콘텐츠가 있는 이유
> 자기 효능감(Self-Efficacy)은 목표 달성의 핵심 동기입니다. 이번 주 얼마나 걸었는지 시각적으로 보여주면 "이만큼 했으니 더 할 수 있다"는 긍정적 피드백을 유발합니다. NRC 스타일로 주간 목표 프로그레스 바를 제거하고, 순수한 산책 통계(거리, 횟수, 시간)에 집중하여 간결한 정보를 제공합니다.

### 섹션 3: 최근 산책 기록

#### 콘텐츠 구성
- 섹션 타이틀 "최근 산책" + "전체보기" (마이페이지 링크)
- 산책 기록 카드 (2-3개)
- 각 카드: 날짜, 코스명, 반려동물, 거리/시간/칼로리, 미니 경로 SVG

#### 이 콘텐츠가 있는 이유
> 과거 기록을 보여줌으로써 "꾸준히 하고 있다"는 성취감을 제공합니다. Variable Ratio Reinforcement(변동 비율 강화)에 따라, 산책할 때마다 다른 결과(다른 경로, 다른 거리)가 기록되는 것 자체가 보상이 됩니다. 주간 요약 바로 다음에 배치하여 "이번 주 요약 → 최근 기록" 흐름으로 산책 활동에 대한 인식을 강화합니다. 카드 클릭 시 산책상세 화면으로 이동하여 상세 정보를 확인할 수 있습니다.

### 섹션 4: 진행 중인 뱃지 (가로 스크롤)

#### 콘텐츠 구성
- 섹션 타이틀 "진행 중인 뱃지" + "전체보기" (뱃지컬렉션 화면 링크)
- 가로 스크롤 카드 (3-4개)
- 각 카드: 카테고리 아이콘 + 뱃지 이름 + 프로그레스 바 + 현재/목표 수치

#### 이 콘텐츠가 있는 이유
> **기존 "챌린지"를 통합 뱃지 시스템으로 변경**한 핵심 섹션입니다. Apple Fitness에서 "업적이 곧 완료됩니다" 알림이 사용자를 운동하게 만드는 것처럼, 진행 중인 뱃지의 프로그레스 바는 Zeigarnik Effect(미완료 과제를 기억하는 심리)를 활용합니다. "38/50km - 76%"처럼 목표 근접 수치를 보여주면 "조금만 더 걸으면 된다"는 완결 욕구가 생깁니다.

### 섹션 5: 내 주변 (지도 + 장소 리스트)

#### 콘텐츠 구성
- 섹션 타이틀 "내 주변" + "지도보기" (지도 탭 링크)
- 미니 지도 (현재 위치 중심, 180px)
- 장소 리스트 (공원/카페/병원 - 아이콘, 이름, 상세, 거리)

#### 이 콘텐츠가 있는 이유
> 산책의 "어디로 갈지" 결정을 도와줍니다. 선택의 역설(Paradox of Choice)을 방지하기 위해 3개 추천 장소만 노출하고, 전체 지도는 지도 탭(Tab 4)에서 확인할 수 있습니다. 반려동물 동반 가능 여부, 동물병원 위치 등 반려동물 특화 정보는 일반 지도앱과의 차별점이며, 공공데이터 API 연동으로 구현합니다.

### 섹션 6: 광고 배너

#### 콘텐츠 구성
- 네이티브 광고 영역 (카카오 AdFit)
- 반려동물 관련 광고 (간식, 용품 등)
- AD 뱃지 표시

#### 이 콘텐츠가 있는 이유
> 무료 사용자를 위한 수익화 수단입니다. 컨텐츠 사이 자연스러운 위치에 네이티브 형태로 배치하여 UX 방해를 최소화합니다. 프리미엄 가입 시 제거되므로, 광고 피로감이 프리미엄 전환 동기로도 작용합니다.

### 섹션 7: 프리미엄 배너

#### 콘텐츠 구성
- 다크 그라데이션 배경 배너
- 왕관 아이콘 + "멍이랑 프리미엄" + 혜택 요약
- 가격 표시 (3,900원/월)

#### 이 콘텐츠가 있는 이유
> 홈 화면 하단에 배치하여 모든 무료 기능을 경험한 후 자연스럽게 프리미엄을 인지하게 합니다. AIDA 모델(Attention -> Interest -> Desire -> Action)의 흐름을 따라, 먼저 무료 기능으로 가치를 경험한 뒤 프리미엄 전환을 유도합니다.

## 4. UX 설계 의도

### 4.1 정보 계층 구조
1. **즉시 행동 유도** (날씨 → 플로팅 산책 버튼): 스크롤 없이 핵심 액션에 접근
2. **활동 현황** (주간 요약 → 최근 산책): 성취감 제공
3. **동기부여** (뱃지 → 내 주변): 목표와 탐색
4. **수익화** (광고 → 프리미엄): 자연스러운 전환 유도

### 4.2 인터랙션 패턴
- 세로 스크롤: 전체 콘텐츠 탐색
- 가로 스크롤: 뱃지 카드 (스크롤 스냅)
- 전체화면 오버레이: 산책 시작 시 (플로팅 버튼 → 반려동물 선택 → 산책)

### 4.3 접근성 고려사항
- 모든 버튼에 `aria-label` 제공
- 색상 대비 WCAG AA 기준 충족
- 이모지 아이콘은 보조적 역할, 텍스트로 의미 전달
- 탭 가능한 최소 영역 44x44px 확보
