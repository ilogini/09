# 뱃지 컬렉션 스크린 - 멍이랑 (withbowwow)

> 기존 "챌린지 스크린"을 대체하는 통합 뱃지 시스템의 핵심 허브 페이지.
> Apple Fitness의 업적 뱃지 + Pikmin Bloom의 수집형 보상을 결합한 설계.
> **칭호(타이틀) 시스템과 연동하여 뱃지 수집과 칭호 획득을 통합 관리한다.**

---

## 1. 스크린 개요

### 1.1 목적

뱃지 컬렉션 스크린은 사용자가 산책 활동을 통해 획득한 모든 뱃지를 한눈에 확인하고, 아직 달성하지 못한 뱃지의 진행 상황을 파악하며, 다음 목표를 설정할 수 있는 중앙 허브 역할을 한다. 기존 기획에서 분리되어 있던 "챌린지 시스템"과 "배지 컬렉션"을 하나의 통합 뱃지 시스템으로 재설계하여, 사용자가 단일 인터페이스에서 모든 성취와 진행 상황을 관리할 수 있도록 한다. **또한 마이페이지의 칭호(타이틀) 시스템과 연동되어, 특정 뱃지 조합이나 카테고리 완성이 칭호 획득 조건에 기여하는 구조를 갖는다.**

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 성취 확인 | "내가 얼마나 모았는지 한눈에 보고 싶다" | 필수 |
| 목표 인지 | "다음 뱃지까지 얼마나 남았는지 알고 싶다" | 필수 |
| 수집 동기 | "아직 잠겨있는 뱃지를 보면 채우고 싶다" | 필수 |
| 칭호 연동 | "이 뱃지를 모으면 어떤 칭호를 얻을 수 있는지 알고 싶다" | 상위 |
| 자랑/공유 | "획득한 뱃지를 친구에게 보여주고 싶다" | 선택 |

### 1.3 진입 경로

```
경로 1: 홈 > "진행 중인 뱃지" 섹션 > "전체보기" 탭
경로 2: 마이페이지 > 등급 카드 하단 > "뱃지 컬렉션" 링크
경로 3: 산책 완료 화면 > 획득 뱃지 알림 > 뱃지 탭
경로 4: 알림 > 뱃지 획득 알림 > 탭하여 이동
경로 5: 마이페이지 > 칭호 컬렉션 > "관련 뱃지 보기" 링크
```

## 2. 화면 구성

### 섹션 1: 상단 뱃지 통계 요약 바

#### 콘텐츠 구성

- **배경**: Primary 컬러(#10B981) 그라데이션 카드, 좌우 패딩 16px
- **총 획득 수 / 전체 수**: "24 / 47 뱃지 수집" 형태의 큰 숫자 텍스트 (28pt Bold)
- **프로그레스 링**: 원형 프로그레스 바 (획득 비율 시각화, 51% 등)
- **최근 획득 뱃지**: 마지막 획득 뱃지 아이콘 + 이름 + 획득일 ("2일 전 획득" 상대 시간)
- **수집 등급 라벨**: 획득 비율에 따른 등급 표시 (초보 수집가 0-20% / 열심 수집가 21-50% / 마스터 수집가 51-80% / 레전드 81-100%)
- **칭호 진행 안내**: "다음 칭호까지 뱃지 3개 더 필요!" (칭호 연동 힌트)

#### 이 콘텐츠가 있는 이유

> 통계 요약을 스크린 최상단에 배치하는 것은 Apple Fitness의 "업적" 탭 진입 시 상단에 총 업적 수를 보여주는 패턴을 참고한 설계이다.
>
> - **엔다우먼트 효과(Endowment Effect)**: 사용자가 이미 보유한 뱃지 수를 먼저 보여줌으로써 "이미 이만큼 모았다"는 소유감을 강화한다.
> - **목표 근접 효과(Goal Gradient Effect)**: "24/47"이라는 숫자는 절반 이상을 달성했다는 인식을 주어, 완전 수집을 향한 가속 행동을 유도한다.
> - **칭호 연동 힌트**: "다음 칭호까지 뱃지 3개 더 필요"라는 메시지는 뱃지 수집과 칭호 획득의 연결고리를 명확히 하여, 뱃지 수집의 추가적인 동기를 부여한다.
> - **비즈니스 관점**: 수집 등급 시스템은 프리미엄 전용 뱃지의 존재를 암시하여 업셀 경로를 형성한다.

### 섹션 2: 카테고리 필터 탭

#### 콘텐츠 구성

- **수평 스크롤 탭 바**: 화면 상단 고정 (Sticky)
- **탭 목록**:
  - 전체 (47)
  - 거리 (8)
  - 연속 (7)
  - 탐험 (5)
  - 시간 (5)
  - 스페셜 (7)
  - 시즌 (4)
- **활성 탭 스타일**: Primary 컬러 밑줄 + Bold + 뱃지 카운트
- **비활성 탭 스타일**: Gray 텍스트 + Regular
- **탭 전환 애니메이션**: 좌우 슬라이드 전환 (300ms ease-in-out)

#### 이 콘텐츠가 있는 이유

> 카테고리 필터 탭은 Nike Run Club의 업적 카테고리 분류와 Apple Fitness의 목표 유형별 탭 분류를 결합한 패턴이다.
>
> - **힉스의 법칙(Hick's Law)**: 47개의 뱃지를 한꺼번에 나열하면 인지 부하가 크게 증가한다. 6개 카테고리로 분류하면 사용자는 한 번에 5-8개의 뱃지만 처리하면 되어 의사결정 시간이 단축된다.
> - **탐색 효율성**: 사용자가 "거리 뱃지를 얼마나 모았지?"라는 구체적 질문을 가지고 진입했을 때, 전체 목록을 스크롤하는 대신 해당 탭을 즉시 탭하여 원하는 정보에 도달할 수 있다.

### 섹션 3: 진행 중인 뱃지 하이라이트

#### 콘텐츠 구성

- **섹션 제목**: "곧 달성! 진행 중인 뱃지" + 아이콘
- **수평 스크롤 카드 리스트** (카드 너비 160px, 높이 200px):
  - **뱃지 아이콘**: 반투명 상태 (opacity 0.7) + 컬러 필터
  - **뱃지 이름**: "50km 클럽" (14pt Medium)
  - **프로그레스 바**: 선형 진행률 바 (높이 6px, Primary 컬러 채움)
  - **진행률 텍스트**: "42.3 / 50km (84%)" (12pt, Gray)
  - **남은 목표**: "7.7km 남음" (12pt, Primary Bold)
- **정렬 기준**: 진행률 높은 순 (달성 임박한 뱃지 우선)
- **최대 표시 수**: 5개 (그 이상은 "더보기"로 전체 목록)
- **카드 탭 시**: 해당 뱃지 상세 바텀시트 열림

#### 이 콘텐츠가 있는 이유

> 이 섹션은 Apple Fitness가 "거의 완료" 상태의 링을 화면 상단에 강조하는 패턴에서 착안했다.
>
> - **목표 근접 효과(Goal Gradient Effect)**: "84% 완료"인 뱃지를 상단에 보여주면 사용자는 "조금만 더 하면 된다"는 심리적 압박감과 동시에 성취 기대감을 느껴 즉시 산책을 나가려는 행동이 유발된다.
> - **자이가르닉 효과(Zeigarnik Effect)**: 미완성 과제는 완성된 과제보다 기억에 더 오래 남는다. 프로그레스 바가 채워지지 않은 상태를 시각적으로 보여줌으로써 사용자는 앱을 닫은 후에도 "남은 7.7km"를 기억하고 다시 돌아온다.
> - **비즈니스 목표**: 진행 중 뱃지 하이라이트는 DAU(일일 활성 사용자)와 리텐션에 직접적으로 기여한다.

#### 칭호 연동 구조

뱃지 수집 현황은 마이페이지의 칭호(타이틀) 시스템과 다음과 같이 연동된다:

| 칭호 | 관련 뱃지 조건 | 설명 |
|------|---------------|------|
| 🌅 얼리버드 | 스페셜 "얼리버드" 뱃지 + 오전 산책 누적 30회 | 뱃지 획득 후 추가 조건 충족 |
| 🦉 올빼미 | 스페셜 "올빼미" 뱃지 + 야간 산책 누적 30회 | 뱃지 획득 후 추가 조건 충족 |
| 🏃 마라토너 | 거리 뱃지 "100km 마스터" 이상 + 10km 단일 산책 5회 | 거리 뱃지 연계 |
| 🧭 탐험가 | 탐험 뱃지 "마스터 탐험가" (50곳 방문) | 탐험 뱃지 카테고리 완성 |
| 🔥 연속왕 | 연속 뱃지 "30일 히어로" 이상 | 연속 뱃지 연계 |
| 📸 사진왕 | 스페셜 "포토그래퍼" 뱃지 + 총 사진 200장 | 뱃지 획득 후 추가 조건 충족 |

> **칭호와 뱃지의 관계**: 뱃지는 단기~중기 성취 단위이고, 칭호는 뱃지들을 기반으로 한 장기 성취 표현이다. 뱃지 컬렉션에서 "이 뱃지를 모으면 칭호에 가까워진다"는 안내를 통해 뱃지 수집의 동기를 강화한다.

#### 이 콘텐츠가 있는 이유

> 3열 그리드 레이아웃은 Apple Fitness 업적 화면의 표준 패턴이며, 뱃지의 시각적 상태 구분(earned/in_progress/locked)은 Pikmin Bloom의 도감 시스템에서 차용했다.
>
> - **수집 충동(Collecting Instinct)**: 인간은 본능적으로 세트를 완성하려는 욕구가 있다. 회색 실루엣의 locked 뱃지는 "빈 칸"으로 인식되어 채우고 싶은 충동을 자극한다.
> - **정렬 전략**: 획득 → 진행중 → 미획득 순서로 배치하는 이유는, 먼저 성취감(earned)을 제공하고, 이어서 현재 목표(in_progress)를 인지시킨 뒤, 마지막으로 미래 목표(locked)를 보여주는 감정 곡선을 설계한 것이다.
> - **칭호 연동 효과**: 카테고리 완성 시 칭호 획득 가능을 안내하면, "거리 뱃지를 다 모으면 마라토너 칭호를 얻을 수 있다"는 상위 목표가 생겨 뱃지 수집 활동이 더욱 의미 있어진다.
> - **비즈니스 연결**: 시즌 뱃지(4개)는 기간 한정이므로 FOMO를 유발하여 해당 시즌 동안의 앱 사용 빈도를 높인다.

### 섹션 5: 뱃지 상세 바텀시트

#### 콘텐츠 구성

- **트리거**: 그리드 내 뱃지 카드 탭
- **바텀시트 높이**: 화면 높이의 50% (드래그로 확장/축소)
- **구성 요소**:
  - **뱃지 아이콘**: 중앙 정렬, 96x96px 크기 (earned 시 글로우 애니메이션)
  - **뱃지 이름**: 24pt Bold, 중앙 정렬
  - **뱃지 설명**: "누적 50km를 산책하면 획득할 수 있습니다" (14pt, Gray)
  - **획득 조건 상세**:
    - earned: "2026.02.08 획득" + 체크 아이콘
    - in_progress: 프로그레스 바 + "42.3 / 50km (84%)" + 예상 달성일
    - locked: 자물쇠 아이콘 + 획득 조건 텍스트
  - **카테고리 태그**: "거리 뱃지" 칩
  - **희귀도 표시**: 전체 사용자 중 획득 비율 ("전체 사용자의 12%가 보유")
  - **칭호 연동 안내**: "이 뱃지는 🏃 마라토너 칭호와 연관됩니다" (해당 시)
  - **공유 버튼**: earned 상태일 때만 활성화 (카카오톡, 인스타그램 스토리)
  - **닫기 핸들**: 상단 중앙 드래그 바

#### 이 콘텐츠가 있는 이유

> 바텀시트 패턴은 iOS Human Interface Guidelines에서 권장하는 모달 인터랙션 방식이다.
>
> - **상세 정보 점진적 공개(Progressive Disclosure)**: 그리드에서는 아이콘과 이름만 보여주고, 탭 시 상세 조건과 진행률을 보여주는 것은 정보 계층화 원칙이다.
> - **희귀도 표시의 심리적 효과**: "전체 사용자의 12%가 보유"라는 정보는 뱃지의 희소성(Scarcity)을 부각하여, 해당 뱃지를 보유한 사용자에게는 우월감을, 미보유 사용자에게는 도전 의욕을 부여한다.
> - **칭호 연동 안내**: 뱃지 상세에서 관련 칭호를 안내하면, 사용자가 "이 뱃지를 얻으면 칭호에 가까워지는구나"를 인지하여 뱃지 수집의 장기적 의미를 이해할 수 있다.
> - **예상 달성일 계산**: in_progress 상태에서 "약 3일 후 달성 예상"을 보여주면, 구체적인 시간 프레임이 목표를 더 현실적으로 만든다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
Level 1 (최상위): 통계 요약 — "나의 전체 진행 상황" + 칭호 연동 힌트
Level 2 (필터링): 카테고리 탭 — "원하는 유형만 보기"
Level 3 (하이라이트): 진행 중 뱃지 — "지금 가장 가까운 목표"
Level 4 (전체 뷰): 뱃지 그리드 — "모든 뱃지 탐색" + 칭호 연동 표시
Level 5 (상세): 바텀시트 — "개별 뱃지 깊이 있는 정보" + 관련 칭호 안내
```

이 계층 구조는 사용자가 "총체적 이해 → 필터링 → 핵심 포커스 → 탐색 → 깊이 있는 정보" 순서로 자연스럽게 정보를 소비하도록 설계되었다. 대부분의 사용자는 Level 3(진행 중 뱃지)까지만 확인하고 이탈하며, 수집 지향적 사용자만 Level 4-5로 진입한다.

### 3.2 인터랙션 패턴

| 인터랙션 | 동작 | 피드백 |
|---------|------|--------|
| 카테고리 탭 전환 | 탭 선택 | 밑줄 슬라이드 애니메이션 + 그리드 교차 페이드 |
| 진행 중 뱃지 카드 스와이프 | 좌우 수평 스크롤 | 관성 스크롤 + 스냅 |
| 뱃지 카드 탭 | 카드 터치 | 카드 스케일 애니메이션(0.95→1.0) + 바텀시트 올라옴 |
| 뱃지 획득 시 | 조건 충족 순간 | 파티클 이펙트 + 글로우 + 햅틱(heavy) + 사운드 |
| 바텀시트 닫기 | 하단 스와이프 또는 외부 탭 | 슬라이드 다운 |
| 공유 버튼 | 탭 | 네이티브 공유 시트 |

### 3.3 접근성 고려사항

- **VoiceOver/TalkBack 지원**: 모든 뱃지 아이콘에 `accessibilityLabel` 적용 (예: "50km 챔피언 뱃지, 진행 중, 84% 완료")
- **색각 이상 대응**: earned/in_progress/locked 상태를 색상만이 아닌 형태(글로우/프로그레스바/자물쇠)로도 구분
- **터치 영역**: 뱃지 카드 최소 터치 영역 44x44pt (iOS HIG 기준)
- **다크 모드**: 배경 #121212, 카드 #1E1E1E, 텍스트 #E0E0E0으로 전환
- **동적 폰트**: 시스템 폰트 크기 설정에 따라 텍스트 크기 조절 (최소 12pt ~ 최대 24pt)
- **애니메이션 감소 모드**: `prefers-reduced-motion` 시 파티클/글로우 효과 비활성화

## 5. 기존 챌린지 시스템과의 차이점

| 항목 | 기존 (챌린지 스크린) | 변경 (뱃지 컬렉션) |
|------|---------------------|-------------------|
| 구조 | 진행중/참여가능/완료 3탭 | 카테고리 6탭 + 통합 그리드 |
| 목표 | 기간 한정 챌린지 | 영구 누적 + 시즌 한정 혼합 |
| 보상 | 포인트 + 뱃지 (분리) | 뱃지 자체가 보상 (통합) |
| 참여 방식 | 수동 참여 신청 | 자동 추적 (별도 참여 불필요) |
| 진행률 | 챌린지별 개별 확인 | 전체 뱃지를 한 화면에서 확인 |
| 동기부여 | 단기 목표 중심 | 단기+중기+장기 목표 혼합 |
| 칭호 연동 | 없음 | 뱃지 수집 → 칭호 획득 연계 |

이 변경으로 사용자는 별도로 챌린지에 "참여"하는 번거로움 없이, 산책만 하면 자동으로 뱃지 진행률이 올라가는 무마찰(Frictionless) 경험을 얻게 된다. 또한 뱃지 수집이 칭호 시스템과 연동되어 장기적인 수집 동기가 강화된다.
