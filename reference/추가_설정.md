# 설정 스크린 - 멍이랑 (withbowwow)

> 마이페이지에서 진입하는 앱 설정 서브페이지.
> 알림, 구독, 계정, 앱 정보를 한 곳에서 관리하는 설정 허브.
> **모든 탭의 공통 헤더 설정 아이콘(톱니바퀴)에서도 직접 진입 가능하다.**

---

## 1. 스크린 개요

### 1.1 목적

설정 스크린은 사용자가 앱의 알림 수신 여부, 구독 상태, 계정 정보, 앱 버전 등 **개인화 및 관리 기능을 한 곳에서 통합적으로 제어**할 수 있는 서브페이지다. 마이페이지의 핵심인 "성장 대시보드(등급/칭호/뱃지)" 영역과 분리하여, 관리 기능이 마이페이지의 핵심 경험을 방해하지 않도록 설계했다. NRC가 프로필 화면의 톱니바퀴 아이콘 뒤에 설정을 숨겨두어 프로필 본연의 역할에 집중하게 하는 패턴을 따른다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 알림 제어 | "산책 리마인더, 뱃지 알림, 소셜 알림을 내 필요에 맞게 켜고 끄고 싶다" | 필수 |
| 구독 확인 | "현재 내 구독 플랜이 뭔지, 프리미엄 혜택이 뭔지 확인하고 싶다" | 필수 |
| 계정 관리 | "연결된 카카오 계정을 확인하고 비밀번호를 변경하고 싶다" | 필수 |
| 앱 정보 확인 | "앱 버전, 개인정보처리방침, 이용약관을 확인하고 싶다" | 선택 |
| 로그아웃 | "계정에서 안전하게 로그아웃하고 싶다" | 필수 |

### 1.3 진입 경로

```
경로 1: 메인페이지(홈) > 상단 헤더 ⚙️ 설정 아이콘 탭
경로 2: 소셜 > 상단 헤더 ⚙️ 설정 아이콘 탭
경로 3: 지도 > 상단 헤더 ⚙️ 설정 아이콘 탭
경로 4: 마이페이지 > 상단 헤더 ⚙️ 설정 아이콘 탭
경로 5: 푸시 알림 > 알림 설정 딥링크 (알림 수신 거부 시 유도)
```

> **공통 헤더에 설정 아이콘을 배치하는 이유**: 기존에는 마이페이지 내부에 설정 섹션이 포함되어 있었으나, NRC 스타일 리디자인에서 마이페이지를 "성장 대시보드"로 집중시키기 위해 설정을 독립 서브페이지로 분리했다. 모든 탭의 공통 헤더에 톱니바퀴 아이콘을 배치하여, 사용자가 어느 화면에서든 한 번의 탭으로 설정에 접근할 수 있다.

## 2. 화면 구성

### 섹션 1: 알림 설정

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "알림 설정" (18pt Bold) |
| 형태 | 리스트 형태의 토글 스위치 그룹 |
| 산책 리마인더 토글 | ON/OFF - 매일 설정 시간에 "산책 갈 시간이에요!" 푸시 알림 |
| 리마인더 시간 설정 | 산책 리마인더 ON 시 노출, 시간 피커 (기본값: 오후 4:00) |
| 뱃지 획득 알림 토글 | ON/OFF - 새 뱃지 획득, 뱃지 진행률 마일스톤 도달 시 알림 |
| 소셜 알림 토글 | ON/OFF - 응원, 챌린지 초대, 클럽 활동, 팔로우 등 소셜 알림 |
| 등급 변동 알림 토글 | ON/OFF - 등급 업, 칭호 획득 시 알림 |

#### 알림 설정 레이아웃

```
┌──────────────────────────────────────┐
│  알림 설정                            │
│                                      │
│  ┌──────────────────────────────┐   │
│  │ 🔔 산책 리마인더        [ON ]│   │
│  │    매일 오후 4:00에 알림     │   │
│  └──────────────────────────────┘   │
│  ┌──────────────────────────────┐   │
│  │ ⏰ 리마인더 시간     [16:00 >]│   │  ← 시간 피커
│  └──────────────────────────────┘   │
│  ┌──────────────────────────────┐   │
│  │ 🏆 뱃지 획득 알림       [ON ]│   │
│  │    뱃지 획득 및 진행 알림    │   │
│  └──────────────────────────────┘   │
│  ┌──────────────────────────────┐   │
│  │ 👥 소셜 알림            [ON ]│   │
│  │    응원, 초대, 클럽 활동     │   │
│  └──────────────────────────────┘   │
│  ┌──────────────────────────────┐   │
│  │ ⭐ 등급/칭호 알림       [ON ]│   │
│  │    등급 업, 칭호 획득        │   │
│  └──────────────────────────────┘   │
│                                      │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 알림 설정을 최상단에 배치하는 것은 iOS 설정 앱과 대부분의 모바일 앱 설정 화면에서 검증된 패턴이다.
>
> - **사용자 자율성(Autonomy)**: Self-Determination Theory(자기결정이론)에 따르면, 사용자가 알림 수신을 스스로 통제할 수 있다는 느낌은 앱에 대한 긍정적 태도를 형성한다. 알림을 강제하면 오히려 앱 삭제로 이어지지만, 세분화된 토글을 제공하면 "필요한 알림만 받겠다"는 선택지를 주어 장기 사용 가능성을 높인다.
> - **산책 리마인더의 행동 설계**: Fogg Behavior Model에서 Trigger(촉발)는 행동을 유발하는 핵심 요소다. 매일 일정 시간에 "산책 갈 시간이에요!"라는 푸시 알림은 Implementation Intention(실행 의도)을 형성하여 산책 습관화를 돕는다. 사용자가 직접 시간을 설정하면 Commitment(약속) 효과도 추가된다.
> - **비즈니스 목표**: 알림은 DAU(일일 활성 사용자)에 직접적으로 기여하는 리텐션 도구다. 다만 과도한 알림은 알림 권한 해제나 앱 삭제로 이어지므로, 세분화된 제어를 통해 사용자가 "적정 알림량"을 스스로 결정하게 한다. 이는 장기 리텐션에 더 유리하다.

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "구독 관리" (18pt Bold) |
| 현재 플랜 표시 | "무료 플랜" 또는 "프리미엄 (월 3,900원)" 상태 카드 |
| 프리미엄 혜택 요약 | 무료 사용자에게 프리미엄 혜택 리스트 노출 |
| CTA 버튼 | 무료: "프리미엄 시작하기" → 프리미엄.html 링크 / 프리미엄: "구독 관리" → 스토어 구독 관리 |
| 결제일 정보 | 프리미엄 사용자만 노출 - "다음 결제일: 2026.03.15" |

#### 이 콘텐츠가 있는 이유

> 구독 관리를 알림 설정 바로 다음에 배치하는 것은, 사용자가 설정 화면을 방문할 때 "관리"의 맥락에서 자연스럽게 구독 상태를 인지하도록 하기 위함이다.
>
> - **업셀 기회(Upsell Opportunity)**: 무료 사용자가 설정 화면을 방문할 때마다 프리미엄 혜택을 요약하여 보여주면, Mere Exposure Effect(단순 노출 효과)에 의해 프리미엄에 대한 친숙도와 호감도가 점진적으로 상승한다. 이는 강제적 팝업보다 부드러운 전환 유도 전략이다.
> - **투명한 구독 관리**: 프리미엄 사용자에게 결제일과 결제 수단을 명확히 표시하는 것은 Apple App Store와 Google Play의 구독 투명성 가이드라인을 따른 것이다. 구독 해지를 숨기지 않고 쉽게 접근할 수 있게 하면 오히려 사용자 신뢰가 높아져 장기 구독 유지율이 상승한다.
> - **비즈니스 목표**: 설정 화면은 프리미엄 전환의 "소프트 터치포인트"다. 매번 설정을 열 때마다 프리미엄 혜택을 인지하게 되므로, 누적 노출이 전환율 상승에 기여한다.

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "계정 설정" (18pt Bold) |
| 연결된 계정 | 로그인 방식 표시 (카카오 로그인 아이콘 + "카카오 계정으로 로그인됨") |
| 이메일 표시 | 연결된 이메일 주소 (마스킹 처리: tes****@kakao.com) |
| 비밀번호 변경 | "비밀번호 변경 >" → 비밀번호 변경 플로우 (소셜 로그인 시 비활성화) |
| 계정 삭제 | "계정 삭제 >" → 탈퇴 확인 다이얼로그 |

#### 계정 삭제 확인 다이얼로그

```
┌──────────────────────────────────────┐
│                                      │
│  정말 탈퇴하시겠어요?                 │
│                                      │
│  탈퇴하면 아래 데이터가 모두           │
│  영구적으로 삭제됩니다:               │
│                                      │
│  • 산책 기록 (187회)                 │
│  • 뱃지 컬렉션 (24개)               │

│  • 반려동물 프로필 및 사진            │
│                                      │

│                                      │

│                    (빨간색)           │
│                                      │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 계정 설정은 사용자의 **보안과 신뢰**에 직결되는 섹션이다.
>
> - **소셜 로그인 우선 전략**: 멍이랑은 카카오 로그인을 기본으로 제공한다. 한국 모바일 앱에서 카카오 로그인은 가장 높은 전환율(약 70%+)을 보이며, 사용자가 별도 비밀번호를 관리할 필요가 없어 가입 장벽을 최소화한다. 카카오 로그인 사용자에게는 비밀번호 변경 항목을 비활성화(grayed out + "카카오 계정은 카카오에서 관리됩니다" 안내)하여 혼란을 방지한다.
> - **계정 삭제의 투명성**: Apple App Store 리뷰 가이드라인 5.1.1에 따르면, 앱 내에서 계정 삭제 기능을 명확히 제공해야 한다. 단, 탈퇴 전 삭제될 데이터를 구체적으로 나열(산책 기록 187회, 뱃지 24개 등)하여 Loss Aversion(손실 회피) 심리를 작동시킨다. "Lv.5 산책전문가 등급이 사라진다"는 정보는 Sunk Cost Effect와 결합하여 이탈 방지에 기여한다.
> - **비즈니스 목표**: 계정 삭제를 숨기지 않되, 삭제 시 잃게 되는 것을 명확히 보여줌으로써 이탈율을 낮춘다. 이는 "투명하지만 전략적인" 리텐션 설계다.

> 앱 정보 섹션은 법적 의무사항과 사용자 신뢰 구축을 위한 필수 구성이다.
>
> - **법적 요구사항**: 개인정보보호법 및 정보통신망법에 따라 개인정보처리방침과 이용약관은 앱 내에서 쉽게 접근 가능해야 한다. Apple App Store와 Google Play 모두 이를 필수 요건으로 명시하고 있다.
> - **버전 표시의 실용성**: 앱 버전을 설정에 표시하면 고객 문의 시 "몇 버전을 사용 중인가요?"라는 질문 없이 사용자가 바로 확인할 수 있다. 또한 앱 업데이트가 필요한 경우 "최신 버전이 아닙니다" 안내를 연결할 수 있다.
> - **문의하기의 리텐션 효과**: 불만이 있는 사용자가 앱을 삭제하기 전에 문의하기를 통해 피드백을 전달할 수 있는 경로를 제공한다. 이는 "불만 → 이탈" 대신 "불만 → 문의 → 해결 → 충성 사용자"의 선순환을 만든다.

### 섹션 5: 로그아웃

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 위치 | 설정 화면 최하단, 모든 섹션 아래 |
| 형태 | 가로 풀너비 Danger 스타일 버튼 |
| 스타일 | 빨간색 텍스트 (#EF4444) + 투명 배경 + 빨간색 테두리 |
| 탭 시 | 로그아웃 확인 다이얼로그 표시 |
| 로그아웃 후 | 로그인/온보딩 화면으로 이동, 로컬 캐시 삭제 |

#### 로그아웃 버튼 및 확인 다이얼로그

```
── 로그아웃 버튼 (화면 최하단) ──
┌──────────────────────────────────────┐
│                                      │
│  ┌──────────────────────────────┐   │
│  │         로그아웃              │   │  ← Danger 스타일 버튼
│  │     (빨간 텍스트 + 테두리)    │   │     (빨강 #EF4444)
│  └──────────────────────────────┘   │
│                                      │
│  멍이랑 v1.0.0                       │  ← 하단 버전 라벨 (Gray)
│                                      │
└──────────────────────────────────────┘

── 로그아웃 확인 다이얼로그 ──
┌──────────────────────────────────────┐
│                                      │
│  로그아웃 하시겠어요?                 │
│                                      │
│  산책 기록과 뱃지는 서버에            │
│  안전하게 보관됩니다.                 │
│  다시 로그인하면 모든 데이터를         │
│  이어서 사용할 수 있어요.             │
│                                      │
│  [취소]           [로그아웃]          │
│                    (빨간색)           │
│                                      │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 로그아웃 버튼을 화면 최하단에 Danger 스타일로 배치하는 것은 iOS/Android 설정 앱의 표준 패턴이다.
>
> - **의도적 접근 어려움**: 로그아웃은 일상적으로 사용하는 기능이 아니므로, 화면 최하단에 배치하여 "실수로 탭"할 가능성을 최소화한다. 이는 Defensive Design(방어적 설계) 원칙이다.
> - **안심 메시지 전략**: 로그아웃 확인 다이얼로그에서 "산책 기록과 뱃지는 서버에 안전하게 보관됩니다"라는 안심 메시지를 제공한다. 이는 사용자가 "로그아웃하면 데이터가 사라지는 건 아닐까?"라는 불안감을 해소하여, 로그아웃 후 재로그인 가능성을 높인다.
> - **Danger 스타일의 시각적 경고**: 빨간색 텍스트와 테두리는 "이 버튼은 일반적인 기능이 아닌 주의가 필요한 액션"이라는 시각적 신호를 전달한다. Color Psychology(색채 심리학)에서 빨간색은 경고와 주의를 의미한다.
> - **비즈니스 목표**: 로그아웃 자체를 막지 않되, "데이터가 안전하다"는 안심 메시지로 재방문 가능성을 유지한다. 로그아웃 후에도 서버에 데이터가 남아있다는 사실은 사용자가 앱을 다시 설치하거나 재로그인할 때 이전 기록을 그대로 이어갈 수 있다는 심리적 안전망이 된다.

## 3. 인터랙션 정의

### 3.1 정보 계층 구조

```
Level 1 (핵심 관리): 알림 설정 — 일상적으로 가장 자주 변경하는 설정
Level 2 (가치 제안): 구독 관리 — 무료/프리미엄 상태 확인 + 업셀
Level 3 (보안 관리): 계정 설정 — 로그인 정보, 비밀번호, 계정 삭제
Level 4 (정보 확인): 앱 정보 — 버전, 약관, 라이선스, 문의
Level 5 (퇴장): 로그아웃 — 최하단 배치, 의도적 접근 필요
```

이 계층 구조는 **"자주 사용 → 가끔 사용 → 거의 사용 안 함"** 순서로 배치한다. 대부분의 사용자는 Level 1(알림 설정)만 조정하고 이탈하며, Level 3-5는 특수한 상황에서만 접근한다. 이는 Pareto Principle(80/20 법칙)에 기반한 설계로, 80%의 사용자가 사용하는 20%의 기능(알림 토글)을 최상단에 배치한다.

### 3.2 인터랙션 패턴

| 인터랙션 | 동작 | 피드백 |
|---------|------|--------|
| 토글 스위치 | 알림 항목 ON/OFF | 토글 애니메이션 + 햅틱(light) + 즉시 서버 반영 |
| 시간 피커 | 리마인더 시간 설정 | iOS 네이티브 시간 피커 바텀시트 |
| 리스트 항목 탭 | 비밀번호 변경, 약관, 문의 등 | 우측 chevron(>) + 새 화면 Push 전환 |
| 프리미엄 CTA 탭 | "프리미엄 시작하기" 버튼 | 프리미엄 소개 화면으로 이동 (모달) |
| 로그아웃 버튼 탭 | 로그아웃 | 확인 다이얼로그 표시 → 확인 시 로그인 화면 |
| 계정 삭제 탭 | 계정 삭제 | 확인 다이얼로그 (삭제될 데이터 상세 표시) |
| 스크롤 | 전체 설정 탐색 | 세로 스크롤, 섹션 간 구분선 |
| 뒤로가기 | 좌측 상단 ← 또는 제스처 | 이전 화면(진입 화면)으로 Pop 전환 |

### 3.3 접근성 고려사항

- **VoiceOver/TalkBack 지원**: 모든 토글 스위치에 `accessibilityLabel` 적용 (예: "산책 리마인더 알림, 현재 켜짐, 이중 탭하여 끄기")
- **색상 대비**: 토글 ON 상태의 Primary 컬러(#10B981)와 배경 간 4.5:1 이상 대비 유지
- **터치 영역**: 토글 스위치 터치 영역 최소 44x44pt, 리스트 항목 높이 최소 48pt
- **Danger 버튼 접근성**: 로그아웃/계정 삭제 버튼에 `accessibilityHint` "이 버튼을 탭하면 확인 화면이 나타납니다" 추가
- **다크 모드**: 배경 #121212, 카드 #1E1E1E, 텍스트 #E0E0E0, Danger 색상 유지 (#EF4444)
- **동적 폰트**: 시스템 폰트 크기 설정에 따라 텍스트 크기 조절, 큰 글씨 모드에서 레이아웃 유지

## 4. 데이터 요구사항

| 데이터 | API/소스 | 갱신 주기 |
|--------|---------|----------|
| 알림 설정 값 | Supabase (user_settings 테이블) | 토글 변경 시 즉시 업데이트 |
| 리마인더 시간 | Supabase (user_settings 테이블) | 시간 변경 시 즉시 업데이트 |
| 구독 상태 | Supabase (user_subscriptions 테이블) + App Store/Play Store | 앱 실행 시 동기화 |
| 결제 정보 | Apple IAP / Google Play Billing | 구독 변경 시 Webhook |
| 연결된 계정 | Supabase (users 테이블 auth_provider) | 고정 (가입 시 결정) |
| 앱 버전 | 앱 번들 내장 (네이티브) | 앱 빌드 시 고정 |
| 개인정보처리방침 | 웹뷰 URL (외부 호스팅) | 법무팀 수정 시 반영 |
| 이용약관 | 웹뷰 URL (외부 호스팅) | 법무팀 수정 시 반영 |

### 알림 설정 저장 로직

```
사용자 토글 변경 이벤트
  ↓
Optimistic UI 업데이트 (즉시 토글 반영)
  ↓
Supabase user_settings 테이블 UPDATE
  ↓
성공 시: 무음 (이미 UI 반영됨)
실패 시: 토글 롤백 + "설정 저장에 실패했습니다" 토스트
  ↓
FCM/APNs 토픽 구독/해제 동기화
  - 산책 리마인더 ON → 로컬 알림 스케줄 등록
  - 산책 리마인더 OFF → 로컬 알림 스케줄 해제
  - 소셜/뱃지/등급 알림 → FCM 토픽 구독/해제
```

### 계정 삭제 처리 로직

```
계정 삭제 요청
  ↓
확인 다이얼로그 표시 (삭제될 데이터 상세)
  ↓
사용자 "탈퇴하기" 탭
  ↓
비밀번호 재확인 또는 소셜 로그인 재인증
  ↓
Supabase Edge Function 호출:
  1. user_settings DELETE
  2. walks DELETE (산책 기록)
  3. walk_photos Storage DELETE (사진)
  4. user_badges DELETE (뱃지)
  5. user_titles DELETE (칭호)
  6. user_grades DELETE (등급)
  7. pets DELETE (반려동물)
  8. users SOFT DELETE (30일 유예 후 Hard Delete)
  ↓
로그아웃 + 로컬 데이터 초기화
  ↓
"계정이 삭제되었습니다" 화면 → 온보딩 화면으로 이동
```

> **30일 유예 기간**: users 테이블은 즉시 삭제하지 않고 `deleted_at` 타임스탬프를 기록하여 30일간 복구 가능한 상태를 유지한다. 30일 이내에 재로그인하면 "계정 복구하시겠습니까?" 안내를 표시한다. 이는 감정적 결정에 의한 탈퇴를 방지하는 안전장치이며, GDPR의 "잊힐 권리"와도 양립한다 (30일 후 완전 삭제).
