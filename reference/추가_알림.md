# 알림 스크린 - 멍이랑 (withbowwow)

> 뱃지 획득, 랭킹 변동, 산책 리마인더, 소셜 활동 등 모든 알림을 통합 관리하는 허브.
> 기존 "챌린지 알림"을 뱃지 기반 알림 시스템으로 전환.

---

## 1. 스크린 개요

### 1.1 목적

알림 스크린은 앱 내에서 발생하는 모든 이벤트를 시간순으로 정리하여 사용자에게 전달하는 중앙 허브이다. 기존 기획의 "챌린지 알림"을 통합 뱃지 시스템에 맞게 "뱃지 획득/진행 알림"으로 전환하고, 랭킹, 소셜, 리마인더, 시스템 알림을 하나의 일관된 인터페이스에서 관리한다. 단순한 알림 목록을 넘어, 사용자의 행동을 유도하고 앱 재방문을 촉진하는 인게이지먼트 도구로 기능한다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 상태 확인 | "뭔가 놓친 게 있는지 빠르게 확인하고 싶다" | ✅ 필수 |
| 성취 알림 | "뱃지를 획득했을 때 즉시 알고 싶다" | ✅ 필수 |
| 동기 부여 | "목표에 가까워지면 격려를 받고 싶다" | ✅ 필수 |
| 소셜 응대 | "누가 팔로우했는지, 초대가 왔는지 알고 싶다" | ⭐ 선택 |

### 1.3 진입 경로

```
경로 1: 모든 화면 > 헤더 우측 > 알림 벨 아이콘 탭
경로 2: 푸시 알림 > 앱 외부에서 탭하여 해당 알림으로 이동
경로 3: 마이페이지 > 설정 > 알림 관리 > "알림 보기" 링크
```

## 2. 화면 구성

### 섹션 1: 알림 헤더 및 관리 도구

#### 콘텐츠 구성

- **네비게이션 바**:
  - 좌측: "← 뒤로가기" 또는 닫기(X) 버튼
  - 중앙: "알림" 제목 (18pt Bold)
  - 우측: "모두 읽음" 텍스트 버튼 (14pt, Primary)
- **필터 탭 바** (선택적):
  - [전체] [뱃지] [랭킹] [소셜] [시스템]
  - 활성 탭: Primary 밑줄 + Bold
  - 각 탭에 미읽음 뱃지 카운트 (빨간 점 또는 숫자)
- **안읽은 알림 카운트**: 헤더 영역에 "새 알림 5건" 표시 (안읽은 알림이 있을 경우)

#### 이 콘텐츠가 있는 이유

> Instagram, Twitter(X), KakaoTalk 등 소셜 앱의 알림 탭은 필터 기능을 통해 알림 유형별 분류를 지원한다. Instagram은 "팔로우 요청" 필터를 별도로 제공한다.
>
> - **인지 부하 관리**: 알림이 쌓이면 중요한 알림을 놓칠 수 있다. 유형별 필터를 제공하면 사용자가 관심 있는 알림 유형만 선택적으로 확인할 수 있어, 정보 과부하를 방지한다.
> - **"모두 읽음" 버튼의 심리적 효과**: 알림 뱃지의 빨간 점은 "제거해야 할 것"으로 인식되어 심리적 불편감을 유발한다(알림 강박). "모두 읽음" 버튼은 이 불편감을 한 번에 해소할 수 있는 수단을 제공하여, 사용자 경험의 부정적 측면을 완화한다.
> - **비즈니스 관점**: 알림 탭 진입 자체가 DAU 카운트에 기여하며, 알림 내용을 확인하는 과정에서 자연스럽게 다른 기능(뱃지 컬렉션, 랭킹, 소셜)으로의 이동이 발생한다.

### 섹션 2: 뱃지 획득 알림

#### 콘텐츠 구성

- **알림 카드 구성**:
  - 좌측: 뱃지 아이콘 (48x48px, 풀컬러 + 글로우)
  - 중앙:
    - 제목: "새 뱃지를 획득했어요!" (15pt Bold)
    - 내용: "🥉 10km 클럽 뱃지를 획득했습니다. 축하합니다!" (14pt, Dark Gray)
    - 시간: "2시간 전" (12pt, Light Gray)
  - 우측: ">" 셰브론 (뱃지 컬렉션으로 이동)
- **카드 스타일**:
  - 안읽음: 좌측에 Primary 컬러 세로 바(3px) + 배경 연한 민트
  - 읽음: 기본 흰색 배경
- **탭 인터랙션**: 탭 시 뱃지 컬렉션 스크린의 해당 뱃지 상세로 딥링크 이동
- **스와이프 인터랙션**: 좌로 스와이프 시 "삭제" 버튼 노출

#### 이 콘텐츠가 있는 이유

> Apple Fitness에서 업적 달성 시 알림 센터에 풀컬러 뱃지 아이콘과 함께 축하 알림을 보내는 패턴을 참고했다. 기존 "챌린지 완료 알림"을 대체한다.
>
> - **강화 이론(Reinforcement Theory)의 즉각적 강화**: 뱃지 획득 알림은 산책 완료 직후에 발생하는 즉각적 보상 신호이다. B.F. 스키너의 조작적 조건화 이론에 따르면, 행동(산책) 직후의 보상(뱃지 알림)은 해당 행동의 반복 가능성을 가장 강하게 높인다.
> - **풀컬러 아이콘의 시각적 보상**: 알림 목록에서 뱃지 아이콘이 유일하게 컬러풀하고 글로우 효과가 있으므로, 다른 알림 사이에서 즉시 눈에 띈다. 이는 시각적 두드러짐(Visual Salience)을 통해 뱃지 획득의 특별함을 강조한다.
> - **챌린지 통합의 이점**: 기존에 "챌린지 완료 알림"과 "뱃지 획득 알림"이 별도로 존재하면 알림이 과도해질 수 있었다. 통합 뱃지 시스템에서는 하나의 알림 유형으로 통합되어 알림 피로도를 줄인다.

### 섹션 3: 뱃지 진행률 알림

#### 콘텐츠 구성

- **알림 카드 구성**:
  - 좌측: 뱃지 아이콘 (48x48px, 반투명 상태)
  - 중앙:
    - 제목: "목표에 거의 다 왔어요!" (15pt Bold)
    - 내용: "🥇 100km 마스터까지 7.7km 남았습니다. 오늘 산책으로 달성해보세요!" (14pt)
    - 프로그레스 바: 인라인 소형 (84%)
    - 시간: "오늘 오전 9:00" (12pt, Light Gray)
  - 우측: "산책 시작" CTA 버튼 (Primary, 소형)
- **트리거 조건**:
  - 진행률 80% 이상일 때 1일 1회 오전 알림
  - 진행률 95% 이상일 때 추가 격려 알림
- **카드 탭 인터랙션**: 산책 시작 화면으로 이동

#### 이 콘텐츠가 있는 이유

> Nike Run Club에서 챌린지 목표에 가까워지면 "Just a little more!"과 같은 격려 푸시를 보내는 패턴을 참고했다.
>
> - **목표 근접 효과(Goal Gradient Effect) 활용**: 목표에 80% 이상 도달한 사용자에게 "거의 다 왔어요"라는 메시지를 보내면, 목표 달성을 위한 마지막 노력을 가속하는 심리적 효과가 발생한다. Columbia University의 커피 스탬프 카드 실험에서 10/12 완성 카드 소유자가 2/12 완성 카드 소유자보다 훨씬 빠르게 스탬프를 채운 것과 동일한 원리이다.
> - **행동 트리거(Behavioral Trigger)**: 알림 내에 "산책 시작" CTA 버튼을 포함하면, 알림 확인에서 실제 행동으로의 전환 마찰(Friction)을 최소화한다. 사용자가 "나중에 해야지"라고 미루는 대신, 즉시 산책을 시작할 수 있는 원클릭 경로를 제공한다.
> - **알림 빈도 조절**: 진행률 80% 이상에서만 발송하고, 1일 1회로 제한하여 알림 피로도를 방지한다. 과도한 알림은 오히려 앱 삭제의 원인이 되므로, 정밀한 트리거 조건 설정이 중요하다.

### 섹션 4: 랭킹 변동 알림

#### 콘텐츠 구성

- **알림 카드 구성**:
  - 좌측: 랭킹 아이콘 (48x48px, 트로피/화살표)
  - 중앙:
    - 순위 상승: "축하합니다! 동네 랭킹 3위로 올라갔어요!" (15pt Bold, Primary)
    - 순위 하락: "아쉽게 랭킹이 4위로 내려갔어요. 다시 도전해보세요!" (15pt Bold, Gray)
    - 현재 순위 요약: "이번 주 2.5km만 더 걸으면 3위 탈환!" (14pt)
    - 시간: "1시간 전" (12pt, Light Gray)
  - 우측: 순위 변동 표시 (↑2 초록 / ↓1 빨강)
- **트리거 조건**:
  - 순위가 변동(상승 또는 하락)될 때마다 발송
  - 주간 랭킹 마감 24시간 전 "마감 임박" 알림
- **탭 인터랙션**: 랭킹 탭으로 이동

#### 이 콘텐츠가 있는 이유

> Strava에서 세그먼트 KOM(King of the Mountain) 순위가 변동되면 푸시 알림을 보내는 패턴을 참고했다.
>
> - **경쟁 심리(Competitive Drive) 활성화**: 순위 하락 알림은 사회적 비교(Social Comparison)를 통한 경쟁 심리를 자극한다. Leon Festinger의 사회적 비교 이론에 따르면, 사람들은 자신과 유사한 타인(동네 산책러)과 비교할 때 가장 강한 동기부여를 받는다. "4위로 내려갔다"는 알림은 "다시 3위를 되찾아야겠다"는 행동을 유발한다.
> - **양방향 감정 설계**: 순위 상승은 긍정적 강화(축하), 순위 하락은 손실 회피(Loss Aversion)를 활용한다. 행동경제학에서 손실의 심리적 무게는 동등한 이익의 약 2배이므로, 순위 하락 알림이 순위 상승 알림보다 더 강한 행동 유도력을 가진다.
> - **"마감 임박" 알림의 긴급성**: 주간 랭킹 마감 24시간 전 알림은 시간적 긴급성(Urgency)을 부여하여, 마지막 순간에 산책을 나가도록 유도한다. 이커머스의 "타임세일 종료 임박" 전략과 동일한 심리적 메커니즘이다.

### 섹션 5: 산책 리마인더

#### 콘텐츠 구성

- **알림 카드 구성**:
  - 좌측: 산책 아이콘 (48x48px, 발자국)
  - 중앙:
    - 제목: "산책 시간이에요!" (15pt Bold)
    - 내용: "초코가 산책을 기다리고 있어요. 오늘 목표: 2km" (14pt)
    - 날씨 정보: "현재 날씨: 맑음 12°C, 산책하기 좋아요!" (12pt, Gray)
    - 시간: "오후 6:00" (12pt, Light Gray)
  - 우측: "산책 시작" CTA 버튼 (Primary)
- **설정 연동**:
  - 마이페이지 > 설정 > 알림 설정에서 시간 설정 가능
  - 기본값: 매일 오후 6시
  - 요일별 개별 설정 가능 (평일/주말 다르게)
- **트리거**: 로컬 알림 (expo-notifications `scheduleNotificationAsync`)
- **스마트 리마인더** (프리미엄):
  - 날씨가 나쁠 때(비, 폭설, 한파) 자동으로 "실내 활동 추천"으로 변경
  - 최근 산책 패턴 분석하여 최적 시간 자동 제안

#### 이 콘텐츠가 있는 이유

> Duolingo의 일일 학습 리마인더는 앱 리텐션의 핵심 도구로, "Don't break your streak!" 메시지로 연속 학습을 유도한다. 멍이랑의 산책 리마인더도 동일한 리텐션 전략을 따른다.
>
> - **습관 루프(Habit Loop) 형성**: Charles Duhigg의 습관의 힘에서 설명하는 습관 루프는 "Cue(신호) → Routine(행동) → Reward(보상)"로 구성된다. 산책 리마인더는 Cue 역할을 하여, 매일 같은 시간에 산책 행동을 트리거한다. 시간이 지나면 리마인더 없이도 자동으로 산책을 나가는 습관이 형성된다.
> - **반려동물의 감정적 트리거**: "초코가 산책을 기다리고 있어요"라는 메시지는 반려동물에 대한 보호자의 책임감과 애정을 자극한다. 이는 단순한 "앱을 열어주세요"보다 훨씬 강력한 감정적 동기부여가 된다.
> - **날씨 정보 포함 이유**: 산책 리마인더에 날씨를 함께 보여주면, 사용자가 별도로 날씨 앱을 확인하는 수고를 줄여준다. "산책하기 좋은 날씨"라는 정보는 "지금이 산책하기에 좋은 타이밍"이라는 추가 동기를 제공한다.

### 섹션 6: 소셜 알림

#### 콘텐츠 구성

- **알림 유형 및 카드 구성**:

| 유형 | 아이콘 | 메시지 예시 | 탭 이동 |
|------|--------|------------|---------|
| 팔로우 | 사람+ 아이콘 | "달이맘님이 회원님을 팔로우합니다" | 해당 프로필 |
| 산책 초대 | 편지 아이콘 | "초코아빠님이 함께 산책을 제안합니다" | 초대 상세 |
| 좋아요 | 하트 아이콘 | "달이맘님이 회원님의 산책 기록을 좋아합니다" | 해당 산책 기록 |
| 댓글 | 말풍선 아이콘 | "초코아빠님이 댓글을 남겼습니다" | 해당 댓글 |
| 챌린지 초대 | 🏅 아이콘 | "초코#1234님이 '주간 30km 도전' 챌린지에 초대했어요" | 참가/거절 |

- **팔로우 알림 특이사항**: 반려동물 프로필 사진 표시 (사람 사진이 아닌 반려동물 사진)
- **산책 초대 알림**: "수락/거절" 액션 버튼 인라인 표시
- **챌린지 초대 알림**: 초대자 프로필(강아지이름#nnnn) + 챌린지 정보(목표거리, 기간) + "참가" / "거절" 인라인 버튼

#### 챌린지 초대 카드

챌린지 초대 알림은 일반 소셜 알림보다 풍부한 정보를 담는 확장형 카드로 표시된다. 초대자의 반려동물 프로필(강아지이름#nnnn 형식), 챌린지 제목, 목표 거리, 참가 기간이 한눈에 보이며, 별도 화면 이동 없이 "참가/거절"을 즉시 결정할 수 있는 인라인 액션 버튼을 포함한다. 이는 알림 확인에서 의사결정까지의 마찰을 최소화하여 챌린지 참가율을 극대화한다.

```
┌──────────────────────────────┐
│ 🏅 챌린지 초대               │
│ 초코#1234님이 초대했어요      │
│ "주간 30km 도전" (2/20~3/6) │
│ [참가]    [거절]             │
│ 1시간 전                     │
└──────────────────────────────┘
```

- **카드 구성 상세**:
  - 좌측 상단: 🏅 챌린지 아이콘 (48x48px, 골드 톤)
  - 제목 행: "챌린지 초대" (15pt Bold)
  - 초대자 정보: "초코#1234님이 초대했어요" (14pt, Dark Gray) - 초대자의 반려동물 이름 + 식별번호
  - 챌린지 정보: 챌린지 이름 + 참가 기간 (14pt, Primary)
  - 인라인 버튼: [참가] (Primary 배경, 흰색 텍스트) / [거절] (Gray 배경, Dark 텍스트)
  - 시간: "1시간 전" (12pt, Light Gray)
- **인터랙션**:
  - "참가" 탭: 버튼 비활성화 + "참가 완료!" 토스트 + 챌린지 상세 화면으로 이동
  - "거절" 탭: 버튼 비활성화 + 카드가 "거절됨" 상태로 전환 (회색 처리)
  - 카드 본문 탭: 챌린지 상세 정보 화면으로 이동

#### 이 콘텐츠가 있는 이유

> Instagram의 활동(Activity) 탭은 팔로우, 좋아요, 댓글 알림을 통합하여 보여주며, 이는 소셜 앱의 표준 알림 패턴이다.
>
> - **소셜 연결감(Social Connectedness)**: 소셜 알림은 "다른 사람들이 내 활동에 관심을 갖고 있다"는 연결감을 제공한다. 자기결정이론(Self-Determination Theory)에서 "관계성(Relatedness)"은 내재적 동기의 3대 요소 중 하나이며, 소셜 알림은 이 관계성을 충족시킨다.
> - **반려동물 중심 소셜**: 팔로우 알림에서 사람 프로필이 아닌 반려동물 프로필 사진을 보여주는 것은 멍이랑의 프라이버시 정책(견주 개인정보 비공개)과 일관된다. 반려동물 사진은 팔로우 수락률을 높이는 효과도 있다.
> - **인라인 액션의 마찰 감소**: 산책 초대에 "수락/거절" 버튼을 알림 내에 포함하면, 별도 화면 이동 없이 즉시 응답할 수 있어 응답률이 높아진다. Slack의 인라인 승인 버튼과 동일한 패턴이다.
> - **챌린지 초대의 소셜 압력(Social Pressure)**: 챌린지 초대 알림은 "초코#1234님이 나를 지목해서 초대했다"는 개인화된 사회적 요청이다. 직접 초대는 일반 공개 챌린지보다 참가율이 약 3배 높다는 것은 게임 산업(예: Pokémon GO 레이드 초대)에서 검증된 패턴이다. 초대자의 반려동물 프로필을 함께 표시하여 감정적 연결을 강화하고, 챌린지 목표/기간을 인라인으로 보여줘 참가 결정에 필요한 정보를 한 화면에서 모두 제공한다.

### 섹션 7: 시스템 알림

#### 콘텐츠 구성

- **알림 유형**:

| 유형 | 아이콘 | 메시지 예시 |
|------|--------|------------|
| 공지사항 | 메가폰 아이콘 | "멍이랑 v2.1 업데이트 안내" |
| 앱 업데이트 | 다운로드 아이콘 | "새 버전이 출시되었습니다. 업데이트하세요!" |
| 이벤트 | 선물 아이콘 | "봄맞이 특별 이벤트가 시작되었습니다!" |
| 시즌 뱃지 오픈 | 뱃지 아이콘 | "겨울왕국 시즌 뱃지가 오픈되었습니다!" |

- **시스템 알림 스타일**: 다른 알림보다 낮은 시각적 강조 (아이콘 회색 톤)
- **탭 인터랙션**: 관련 페이지 또는 웹뷰로 이동

#### 이 콘텐츠가 있는 이유

> 시스템 알림은 앱의 중요 공지를 사용자에게 전달하는 채널이며, 모든 앱이 필수적으로 제공하는 기능이다.
>
> - **시즌 뱃지 알림의 FOMO 효과**: "겨울왕국 시즌 뱃지가 오픈되었습니다"라는 알림은 기간 한정 뱃지의 존재를 인지시켜, 해당 시즌 동안 앱 사용 빈도를 높인다. FOMO(Fear Of Missing Out)는 기간 한정 이벤트에서 가장 강력하게 작용한다.
> - **낮은 시각적 강조의 이유**: 시스템 알림은 사용자 행동과 직접 관련이 없으므로, 뱃지/랭킹/소셜 알림보다 낮은 시각적 우선순위를 부여하여 중요한 알림이 묻히지 않도록 한다.
> - **이벤트 알림의 프리미엄 연결**: 시즌 이벤트를 알리면서 프리미엄 전용 뱃지나 혜택을 함께 노출하여 구독 전환 기회를 창출한다.

### 섹션 8: 날짜별 그룹핑 및 상태 관리

#### 콘텐츠 구성

- **날짜 구분선**: "오늘", "어제", "이번 주", "2026년 2월 1주" 등
- **읽음/안읽음 상태**:
  - 안읽음: 좌측 Primary 세로 바(3px) + 연한 배경색
  - 읽음: 기본 흰색 배경
  - 전체 읽음 처리: 헤더의 "모두 읽음" 버튼
- **빈 상태**: 알림이 없을 경우 "아직 알림이 없어요. 산책을 시작하면 알림이 도착합니다!" + 벨 아이콘 일러스트
- **알림 삭제**: 좌로 스와이프 시 빨간 "삭제" 버튼 노출
- **알림 보존 기간**: 30일 (이후 자동 삭제)
- **무한 스크롤**: 최초 20개 로딩 후 스크롤 시 추가 로딩

#### 이 콘텐츠가 있는 이유

> iOS 알림 센터의 날짜별 그룹핑은 시간적 맥락을 제공하여 알림의 긴급성을 즉시 판단할 수 있게 한다.
>
> - **시간적 맥락의 중요성**: "오늘" 그룹의 알림은 즉시 확인해야 할 것으로, "이번 주" 그룹의 알림은 참고용으로 인식된다. 이 분류가 없으면 사용자는 모든 알림을 동등하게 처리해야 하므로 인지 부하가 증가한다.
> - **안읽음 시각적 표시**: 좌측 세로 바 패턴은 Gmail, Slack 등에서 검증된 "안읽음 표시" 방식이다. 사용자가 목록을 빠르게 스캔할 때 안읽은 항목을 즉시 식별할 수 있다.
> - **30일 보존 정책**: 오래된 알림을 무한정 보관하면 목록이 길어져 최근 알림 접근이 어려워진다. 30일 보존은 충분한 기간이면서도 데이터 관리에 효율적인 기간이다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
Level 1 (관리): 헤더 + 모두 읽음 — 전체 알림 상태 관리
Level 2 (필터): 카테고리 탭 — 관심 유형만 필터링
Level 3 (시간순): 날짜 그룹 — 시간적 맥락 제공
Level 4 (개별): 알림 카드 — 개별 이벤트 내용 확인
Level 5 (행동): CTA / 탭 이동 — 관련 화면으로 전환
```

알림 스크린의 정보 계층은 "관리 → 필터 → 스캔 → 확인 → 행동"의 흐름을 따른다. 대부분의 사용자는 알림 목록을 빠르게 스캔(Level 3-4)하고, 관심 있는 알림만 탭하여 행동(Level 5)으로 이어진다.

### 3.2 인터랙션 패턴

| 인터랙션 | 동작 | 피드백 |
|---------|------|--------|
| 알림 카드 탭 | 카드 터치 | 읽음 처리 + 관련 화면 이동 (push navigation) |
| 좌 스와이프 | 카드 좌로 드래그 | "삭제" 빨간 버튼 노출 |
| 모두 읽음 | 헤더 버튼 탭 | 모든 안읽음 표시 제거 + 확인 토스트 |
| 필터 탭 | 탭 터치 | 해당 유형 알림만 필터링 + 목록 교차 페이드 |
| CTA 버튼 | "산책 시작" 탭 | 산책 준비 화면으로 즉시 이동 |
| 초대 수락/거절 | 인라인 버튼 탭 | 버튼 비활성화 + 결과 토스트 |
| 당겨서 새로고침 | 상단 풀 다운 | 로딩 인디케이터 + 새 알림 로딩 |
| 무한 스크롤 | 하단 도달 | 스피너 + 추가 알림 20개 로딩 |

### 3.3 접근성 고려사항

- **VoiceOver/TalkBack**: 각 알림 카드에 `accessibilityLabel` 적용 (유형 + 내용 + 시간 + 읽음 상태)
- **안읽음 표시**: 색상(세로 바)과 텍스트("새 알림")를 병행하여 색각 이상 사용자도 인지 가능
- **스와이프 대체**: 스와이프 제스처 외에도 롱프레스 메뉴에서 삭제 가능 (운동장애 사용자 고려)
- **다크 모드**: 안읽음 배경색, 카드 배경, 텍스트 색상 다크 테마 전환
- **알림 진동**: 중요 알림(뱃지 획득) 수신 시 햅틱 피드백
