# 홈 화면 (Tab 1) - 멍이랑 (withbowwow)

> 앱을 열었을 때 가장 먼저 마주하는 화면.
> 사용자가 "오늘 산책해야겠다"는 결심을 3초 안에 하도록 설계한다.

---

## 1. 페이지 개요

### 1.1 목적

홈 화면은 멍이랑 앱의 **첫인상이자 행동 트리거**다. 사용자가 앱을 열었을 때 (1) 오늘의 산책 조건을 즉시 파악하고, (2) 현재 진행 상황을 한눈에 확인하며, (3) 하단 중앙 플로팅 버튼을 통해 산책을 시작한다. Nike Run Club이 앱을 열면 간결한 대시보드와 "Start" 버튼이 즉각적인 행동을 이끌어내는 것과 동일한 원리를 적용한다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 오늘 산책 가능한지 확인 | 날씨, 미세먼지 등 외부 조건 판단 | 최상위 |
| 이번 주 얼마나 했는지 | 주간 산책 통계 확인 | 상위 |
| 바로 산책 시작 | 플로팅 중앙 버튼으로 산책 트래킹 진입 | 최상위 |
| 동기부여 요소 확인 | 뱃지 진행률, 최근 기록 등 | 중위 |
| 주변 정보 탐색 | 산책 코스, 동물병원, 반려동물 카페 | 중위 |

### 1.3 진입 경로

| 경로 | 빈도 | 컨텍스트 |
|------|------|---------|
| 앱 최초 실행 | 매번 | 하단 탭 바의 기본 선택 탭 |
| 다른 탭에서 홈 탭 탭 | 높음 | 홈으로 돌아오기 |
| 푸시 알림 탭 → 딥링크 | 중간 | 산책 리마인더, 뱃지 달성 알림 |
| 위젯 탭 | 낮음 | iOS/Android 홈 위젯에서 진입 |

## 2. 화면 구성

### 섹션 0: 앱 헤더 (App Header)

#### 콘텐츠 구성

| 요소 | 위치 | 상세 |
|------|------|------|
| 앱 로고 + 텍스트 | 좌측 | "멍이랑" 로고 (발바닥 아이콘 + 텍스트) |
| 알림 벨 아이콘 | 우측 #1 | 읽지 않은 알림 시 빨간 뱃지 표시 |
| 설정 아이콘 | 우측 #2 | 톱니바퀴 아이콘 → 설정 스크린 |

#### 이 콘텐츠가 있는 이유

> 앱 헤더는 사용자에게 "지금 어떤 앱에 있는지"를 즉각적으로 인지시키는 브랜드 앵커 역할을 한다. NRC 스타일의 미니멀한 헤더로, 기존 GPS 뱃지는 제거하고 알림과 설정만 배치한다.
>
> - **알림 벨**: 챌린지 초대, 뱃지 달성, 소셜 활동 등의 알림을 놓치지 않도록 한다. 빨간 뱃지(unread indicator)는 Zeigarnik 효과를 활용하여 사용자의 탭 행동을 유도한다.
> - **설정 아이콘**: 기존 마이페이지 내 설정 섹션을 헤더로 분리하여, 어느 탭에서든 설정에 바로 접근할 수 있도록 한다. NRC도 상단에 설정 접근을 제공한다.

---

### 섹션 1: 날씨 위젯 (Weather Widget)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 날씨 아이콘 | 맑음/흐림/비/눈 등 상태 아이콘 |
| 현재 기온 | "12도" 형태, 체감온도 병기 |
| 산책 적합도 메시지 | "산책하기 좋은 날이에요!", "미세먼지 주의!" 등 |
| 미세먼지 등급 | 좋음/보통/나쁨/매우나쁨 (색상 코딩) |
| 강수 확률 | "오후 3시 이후 비 예보" 등 시간대 표기 |
| 산책 추천 시간대 | "오전 10시~12시가 최적이에요" |

#### 데이터 소스

```
기상청 단기예보 API: 기온(T1H), 강수형태(PTY), 강수확률(POP)
에어코리아 API: 미세먼지(PM10), 초미세먼지(PM2.5)
```

#### 산책 적합도 판정 로직

| 조건 | 판정 | 메시지 색상 |
|------|------|-----------|
| 맑음 + 미세먼지 좋음 + 10~25도 | 최적 | 초록 (#10B981) |
| 흐림 또는 미세먼지 보통 | 보통 | 노랑 (#F59E0B) |
| 비/눈 또는 미세먼지 나쁨 또는 0도 이하/35도 이상 | 주의 | 빨강 (#EF4444) |

#### 이 콘텐츠가 있는 이유

> 반려동물 산책에서 날씨는 의사결정의 **가장 첫 번째 변수**다. 사용자가 앱을 여는 순간 "오늘 산책 가능한가?"를 즉시 판단할 수 있어야 한다.

---

### 섹션 2: 이번 주 산책 요약 카드 (Weekly Summary Card)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 카드 배경 | 라운드 카드 (border-radius: 16px), 그라데이션 배경 |
| 주간 총 거리 | "12.5 km" (큰 숫자, Bold) |
| 주간 총 횟수 | "3회" |
| 주간 총 시간 | "2시간 15분" |
| 요일별 미니 그래프 | 월~일 7칸 막대 (산책한 날 컬러, 안 한 날 회색) |

#### 이 콘텐츠가 있는 이유

> 주간 요약 카드는 사용자의 **자기효능감(self-efficacy)**을 자극하는 핵심 요소다. "이번 주에 이미 12.5km를 걸었다"는 성취감이 "오늘도 해야지"라는 연속 동기를 만든다. NRC 스타일로 주간 목표 프로그레스 바와 주간 목표 설정 기능은 제거하고, 순수한 활동 통계에 집중하여 간결함을 유지한다.

---

### 섹션 3: 최근 산책 기록 (Recent Walk History)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "최근 산책" + "전체보기 >" 링크 |
| 리스트 형태 | 수직 리스트, 최근 3건 표시 |
| 각 항목 구성 | 날짜 + 반려동물 이름 + 미니 경로맵 + 거리 + 시간 |
| 빈 상태 | "아직 산책 기록이 없어요. 첫 산책을 시작해보세요!" |
| 탭 액션 | 항목 탭 → 산책 상세 스크린 |

#### 각 기록 카드 레이아웃

```
┌──────────────────────────────────┐
│  [미니맵]  2월 11일 (화) 오후 3시  │
│            초코와 함께             │
│            2.34km · 32분 · 128kcal│
└──────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 최근 산책 기록은 사용자의 **활동 이력**을 상기시켜 지속적인 참여를 유도한다. 주간 요약 바로 다음에 배치하여 "이번 주 요약 → 구체적 기록"의 자연스러운 정보 흐름을 형성한다. Sunk Cost Effect(매몰비용 효과)와 Consistency Principle(일관성 원리)이 작용하여 "어제도 산책했으니 오늘도 해야지"라는 심리가 강화된다.

---

### 섹션 4: 진행 중인 뱃지 미리보기 (In-Progress Badges Preview)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "진행 중인 뱃지" + "전체보기 >" 링크 |
| 카드 레이아웃 | 가로 스크롤 (Horizontal ScrollView) |
| 카드 크기 | 120 x 150px 고정 |
| 카드 내용 | 뱃지 아이콘(반투명) + 이름 + 프로그레스 바 + "3.2km / 5km" |
| 최대 표시 수 | 3~5개 (가로 스크롤) |
| 빈 상태 | "새로운 뱃지에 도전해보세요!" |

#### 이 콘텐츠가 있는 이유

> **기존 "챌린지"를 통합 뱃지 시스템으로 변경**한 핵심 섹션이다. Zeigarnik 효과에 따르면 미완료된 과제는 완료된 과제보다 기억에 더 오래 남는다. "3.2km / 5km"처럼 미완성 상태를 보여주면 사용자는 이를 완료하고 싶은 충동을 느낀다.

---

### 섹션 5: 내 주변 (Nearby - Map + Places)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 섹션 헤더 | "내 주변" + "지도보기 >" 링크 (지도 탭으로 이동) |
| 미니 지도 | 현재 위치 중심 반경 1km 지도 (높이 150px) |
| 장소 카테고리 필터 | 전체 / 산책코스 / 동물병원 / 반려카페 / 공원 |
| 장소 리스트 | 카드형 가로 스크롤 (이미지 + 이름 + 거리 + 별점) |
| 최대 표시 수 | 5개 (가로 스크롤) |

#### 이 콘텐츠가 있는 이유

> "내 주변" 섹션은 산책의 **목적지**를 제공하여 무목적 산책을 목적 있는 탐험으로 전환한다. "지도보기" 링크는 지도 탭(Tab 4)으로 연결되어 전체 지도 탐색으로 확장된다.

### 섹션 6: 광고 배너 (Ad Banner)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 배너 형태 | 네이티브 광고 카드 (320 x 100px) |
| 광고 플랫폼 | 카카오 AdFit |
| "광고" 라벨 | 좌상단에 "AD" 또는 "광고" 텍스트 표시 |
| 닫기 버튼 | 우상단 X 버튼 (해당 세션에서만 숨김) |
| 프리미엄 사용자 | 광고 미표시 |

#### 이 콘텐츠가 있는 이유

> 광고는 멍이랑의 핵심 수익 모델(CPM 1,000~3,000원)이다. 핵심 기능 사용에는 방해를 주지 않으면서 자연스러운 스크롤 흐름에 녹아들도록 배치한다.

---

### 섹션 7: 프리미엄 배너 (Premium Promotion Banner)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 배너 형태 | 풀 너비 카드, 그라데이션 배경 (Primary → Secondary) |
| 헤드라인 | "멍이랑 프리미엄으로 업그레이드" |
| 서브텍스트 | "광고 제거, 상세 통계, 무제한 뱃지 트래킹" |
| CTA 버튼 | "무료 체험 시작" |
| 닫기 버튼 | X 버튼 (7일간 숨김) |
| 조건 | 무료 사용자에게만 표시 |

#### 이 콘텐츠가 있는 이유

> 프리미엄 배너는 **수익화 퍼널(monetization funnel)**의 첫 번째 터치포인트다. 홈 화면 하단에 배치하여 핵심 기능 사용을 방해하지 않으면서도, 스크롤을 끝까지 하는 적극적 사용자에게 노출된다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
[1순위 - 즉각 행동]
  └── 날씨 위젯 (산책 가능 여부 즉시 판단)
  └── 플로팅 산책 버튼 (핵심 CTA - 하단 탭 바 중앙)

[2순위 - 활동 현황]
  └── 주간 요약 카드 (자기효능감)
  └── 최근 산책 기록 (활동 이력)

[3순위 - 동기부여]
  └── 진행 중 뱃지 (미완료 과제 효과)
  └── 내 주변 (목적지 제안)

[4순위 - 수익화]
  └── 광고/프리미엄 (수익화)
```

핵심 원칙: **위에서 아래로 갈수록 행동 촉구 → 현황 확인 → 동기부여 → 수익화 순서로 배치**한다. 산책 시작은 하단 플로팅 버튼을 통해 어디서든 접근 가능하다.

### 3.2 인터랙션 패턴

| 패턴 | 적용 위치 | 설명 |
|------|----------|------|
| Pull-to-Refresh | 전체 화면 | 아래로 당겨 데이터 새로고침 (날씨 등) |
| Horizontal Scroll | 뱃지 미리보기, 내 주변 장소 | 가로 스크롤로 더 많은 항목 탐색 |
| Tap to Navigate | 전체보기 링크, 기록 항목 | 탭하여 상세 화면으로 이동 |
| Skeleton Loading | 데이터 로딩 중 | API 응답 대기 중 뼈대 UI 표시 |
| Haptic Feedback | 플로팅 산책 버튼 탭 | 가벼운 진동으로 행동 확인 |
| Lottie Animation | 뱃지 달성 시 | 축하 애니메이션 오버레이 |

### 3.3 접근성 고려사항

| 항목 | 대응 |
|------|------|
| 색상 대비 | 모든 텍스트 WCAG AA 기준 충족 (4.5:1 이상) |
| 터치 영역 | 모든 인터랙티브 요소 최소 44x44px |
| 스크린 리더 | 모든 이미지에 alt 텍스트, 뱃지 진행률 aria-label |
| 다크 모드 | 시스템 설정 연동, 컬러 팔레트 별도 정의 |
| 글꼴 크기 | 시스템 Dynamic Type 지원 (iOS) / Font Scale (Android) |
| 모션 감소 | Reduce Motion 설정 시 애니메이션 비활성화 |
