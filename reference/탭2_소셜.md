# 소셜 화면 (Tab 2) - 멍이랑 (withbowwow)

> Nike Run Club의 소셜 기능을 반려동물 산책에 적용. 친구 간 경쟁, 응원, 클럽 활동.
> NRC의 피드, 챌린지, 클럽, 가이드 런을 반려동물 산책에 맞게 재설계한다.

---

## 1. 페이지 개요

### 1.1 목적

소셜 탭은 멍이랑 앱의 **커뮤니티 허브이자 동기부여 엔진**이다. Nike Run Club의 소셜 기능(피드, 챌린지, 클럽, 가이드 런)을 반려동물 산책에 맞게 재설계하여 4개의 서브 탭으로 구성한다. (1) 피드에서 친구들의 산책 기록을 확인하고 응원하며, (2) 챌린지에서 친구와 경쟁하고, (3) 클럽에서 소속감을 형성하며, (4) 가이드 산책으로 새로운 산책 경험을 발견한다. 기존 랭킹 탭이 지도 탭으로 전환됨에 따라, 주간 리더보드는 이 소셜 탭의 피드 상단에 통합되었다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 친구 산책 활동 확인 | 팔로잉한 친구들의 산책 기록 확인 | 최상위 |
| 응원 및 리액션 | 친구의 산책에 이모지 반응, 응원 보내기 | 상위 |
| 친구와 경쟁 | 주간 거리, 연속 산책 등 챌린지 참여 | 상위 |
| 산책 클럽 활동 | 동네/견종별 클럽에 소속되어 함께 활동 | 중위 |
| 새로운 산책 코스 발견 | 가이드 산책 프로그램으로 새 코스 경험 | 중위 |

### 1.3 진입 경로

| 경로 | 빈도 | 컨텍스트 |
|------|------|---------|
| 하단 탭 바 "소셜" 탭 | 높음 | 일상적 소셜 피드 확인 |
| 푸시 알림 (응원, 챌린지 초대) | 높음 | "초코맘이 응원을 보냈어요!" 딥링크 |
| 홈 → 주간 리더보드 → "전체보기" | 중간 | 홈에서 리더보드 확인 후 상세 진입 |
| 산책 완료 → 공유 → 소셜 피드 | 중간 | 산책 기록 공유 후 피드 확인 |
| 홈 → 가이드 산책 추천 | 낮음 | 홈에서 추천 가이드 산책 진입 |

## 2. 화면 구성

### 공통 헤더 (App Header)

#### 콘텐츠 구성

| 요소 | 위치 | 상세 |
|------|------|------|
| 앱 로고 + 텍스트 | 좌측 | "멍이랑" 로고 (발바닥 아이콘 + 텍스트) |

> 공통 헤더는 모든 탭에서 동일한 구조를 유지한다. NRC가 상단에 심플한 로고와 알림만 배치하듯, 멍이랑도 미니멀한 헤더로 아래 콘텐츠에 최대한의 공간을 할당한다.

### 탭 네비게이션 (NRC 스타일 서브 탭)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 형태 | 수평 탭 바 (헤더 하단 고정) |
| 탭 수 | 4개: 피드 / 챌린지 / 클럽 / 가이드 산책 |
| 활성 상태 | Bold 텍스트 + 하단 인디케이터 라인 (Primary 컬러) |
| 비활성 상태 | Regular 텍스트 + 회색 |
| 전환 방식 | 탭 또는 좌우 스와이프 |
| 뱃지 표시 | 챌린지: 새 초대 수, 클럽: 새 활동 수 |

```
  [피드]    [챌린지 ②]    [클럽]    [가이드 산책]
  ━━━━━━
  (활성)
```

#### 이 콘텐츠가 있는 이유

> NRC는 소셜 기능을 여러 서브 섹션으로 나누어 제공한다. 이 4-탭 구조는 NRC의 Activity / Challenges / Club / Guided Runs 구조를 직접 차용한 것이다.
>
> - **사용자 행동 심리학**: Recognition over Recall(재인 우선) 원칙에 의해 4개의 명확한 탭 라벨은 사용자가 원하는 기능을 빠르게 인지하고 접근하게 한다. 스와이프 전환 지원은 Exploratory Browsing(탐색적 브라우징)을 유도하여 "피드만 보다가 챌린지도 확인해볼까"라는 자연스러운 탐색을 만든다.
> - **비즈니스 목표**: 4개 탭은 각각 다른 사용자 인게이지먼트 패턴을 수집한다. 피드=DAU/체류시간, 챌린지=리텐션/재방문, 클럽=커뮤니티Lock-in, 가이드산책=콘텐츠소비. 다양한 인게이지먼트 채널이 전체 리텐션을 안정적으로 지탱한다.
> - **벤치마킹**: NRC의 상단 탭 구조, Strava의 Feed/Clubs/Challenges 탭, Instagram의 피드/릴스/검색 탭 패턴을 참고한다.

### 피드 탭 (Activity Feed)

#### 1. 주간 리더보드 (Weekly Leaderboard - Compact Card)

| 요소 | 상세 |
|------|------|
| 위치 | 피드 최상단 고정 |
| 형태 | 컴팩트 카드 (가로 풀너비, 높이 120px) |
| 기간 | 이번 주 (월~일) 자동 리셋 |
| 내용 | 내 순위 + TOP 3 썸네일 + 내 기록 |
| 탭 액션 | 리더보드 상세 바텀시트 |
| 순위 변동 | "상승3" 등 지난 주 대비 변동 표시 |

```
┌──────────────────────────────────────┐
│  이번 주 리더보드                       │
│  1위 뽀삐아빠 52km  2위 초코맘 48km    │
│  나: #7 (18.5km) 상승3  > 전체보기    │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 기존 랭킹 전용 탭(Tab 4)이 지도 탭으로 전환됨에 따라, 주간 리더보드를 소셜 피드 상단에 컴팩트 카드로 통합했다.
>
> - **사용자 행동 심리학**: NRC에서 소셜 피드 상단에 주간 마일리지 리더보드를 배치하는 패턴을 차용했다. Social Comparison Theory(사회 비교 이론)에 의해 "친구들 중 내 순위"를 피드 맥락에서 자연스럽게 확인하면 경쟁 동기가 산책 행동으로 바로 연결된다. 컴팩트 형태로 핵심 정보(내 순위, TOP 3, 변동)만 보여주어 인지 부하를 최소화한다.
> - **비즈니스 목표**: 리더보드를 피드 상단에 배치하면 소셜 탭 방문 시 매번 순위를 확인하게 되어 산책 빈도가 증가한다. 별도 탭일 때보다 노출 빈도가 높아 경쟁 효과가 극대화된다.

#### 2. 활동 피드 (Activity Feed Cards)

| 요소 | 상세 |
|------|------|
| 피드 형태 | 수직 무한 스크롤 (Infinite Scroll) |
| 피드 필터 | [전체] / [팔로잉] / [동네] 세그먼트 |
| 실시간 표시 | 현재 산책 중인 친구 상단 표시 ("산책 중") |
| 카드 구성 | 프로필 헤더 + 경로 맵 + 산책 통계 + 사진(선택) + 리액션 바 |

```

│ [프로필] 초코 · 성동구 · 2시간 전 │  ← 프로필 헤더
│                                  │     라이브 산책 중 (해당 시)
├──────────────────────────────────┤
│ ┌──────────────────────────────┐ │
│ │                              │ │
│ │      [산책 경로 맵 이미지]     │ │  ← 경로 지도 캡처
│ │                              │ │
│ └──────────────────────────────┘ │
├──────────────────────────────────┤
│  3.2km  45분  186kcal            │  ← 산책 통계
├──────────────────────────────────┤
│ [사진] [사진] [사진]              │  ← 산책 중 사진 (선택)
├──────────────────────────────────┤
│ 박수 불꽃 하트 엄지 강아지  +12   │  ← 이모지 리액션 카운트
```

| 이모지 | 의미 | 사용 맥락 |

| 불꽃 | 대단해! | 긴 거리, 높은 성과 |
| 하트 | 좋아요 | 귀여운 사진, 감동적 산책 |
| 엄지 | 좋아 | 간단한 긍정 반응 |
| 강아지 | 귀여워! | 반려동물 사진에 대한 반응 |

#### "응원하기" 버튼

| 요소 | 상세 |
|------|------|
| 기본 상태 | "응원하기" 텍스트 버튼 |
| 탭 액션 | 상대방에게 응원 푸시 알림 전송 |
| 탭 후 상태 | "응원 보냄" (재탭 불가) |
| 알림 메시지 | "초코아빠가 당신의 산책을 응원합니다!" |

#### 라이브 산책 인디케이터

| 요소 | 상세 |
|------|------|
| 위치 | 피드 최상단 (리더보드 바로 아래) |
| 형태 | 가로 스크롤 원형 아바타 + "산책 중" 라벨 |
| 탭 액션 | 해당 친구의 실시간 산책 경로 보기 (동의한 경우) |
| 업데이트 | 실시간 (WebSocket / Supabase Realtime) |

```
지금 산책 중
[아바타]  [아바타]  [아바타]
초코     달이     몽이
```

#### 이 콘텐츠가 있는 이유

> 활동 피드는 소셜 탭의 **핵심 콘텐츠 영역이자 앱 체류 시간의 주요 드라이버**다.
>
> - **사용자 행동 심리학**: NRC의 Activity Feed에서 친구의 러닝 기록을 보고 "Cheer" 버튼을 누르는 패턴을 차용했다. 이모지 리액션은 Instagram, Slack, iMessage 등에서 검증된 Low-friction Interaction(저마찰 인터랙션)으로, "좋아요"보다 더 다양하고 세밀한 감정 표현이 가능하다. Social Facilitation(사회적 촉진)에 의해 친구들의 산책 기록을 보면 자신도 산책하고 싶어진다.
> - **"응원하기" 버튼**: NRC의 "Cheer" 기능의 직접 차용이다. 단순 좋아요와 달리, "응원"은 의도적인 행위로 인식되어 받는 사람의 동기부여 효과가 3배 이상 높다(Positive Reinforcement). 응원을 보내면 푸시 알림이 가므로, 앱 재방문의 트리거 역할도 한다.
> - **라이브 산책 인디케이터**: Instagram Stories의 원형 아바타 UX를 차용했다. "지금 이 순간 친구가 산책 중"이라는 실시간 정보는 FOMO(Fear of Missing Out)를 극대화하여 "나도 나가야겠다"는 즉각적 행동을 유도한다.
> - **비즈니스 목표**: 피드는 체류 시간의 핵심 드라이버다. 이모지 리액션, 응원, 댓글 등 인터랙션이 많을수록 상호 피드 콘텐츠가 풍부해지고, 이는 DAU/체류시간 증가의 선순환을 만든다. 피드 사이 네이티브 광고 삽입도 가능하다.

### 챌린지 탭 (Challenges)

#### 1. 진행 중인 챌린지 (Active Challenges)

| 요소 | 상세 |
|------|------|
| 형태 | 카드형 리스트 (수직 스크롤) |
| 챌린지 유형 | 친구 대결 (1:1) / 그룹 챌린지 / 개인 목표 |
| 카드 내용 | 챌린지명 + 참가자 프로필 + 진행률 + 남은 기간 |
| 상태 표시 | 선두/추격 여부, 현재 순위 |

#### 챌린지 유형 상세

| 유형 | 설명 | 참가 인원 | 기간 |
|------|------|----------|------|
| 주간 거리 대결 | 이번 주 산책 거리 경쟁 | 2~10명 | 월~일 (7일) |
| 30일 연속 산책 | 30일 연속 산책 달성 | 1~무제한 | 30일 |
| 월간 목표 | 이번 달 목표 거리 함께 달성 | 2~20명 | 한 달 |
| 주간 횟수 대결 | 이번 주 산책 횟수 경쟁 | 2~10명 | 월~일 (7일) |

#### 진행 중 챌린지 카드 레이아웃

```
┌──────────────────────────────────────┐
│  주간 거리 대결                        │
│  vs 초코맘, 달이아빠 외 2명            │
│                                      │
│  나: 18.5km ============-- 67%       │  ← 내 진행률
│  초코맘: 22.1km =============- 80%   │  ← 1위
│  달이: 15.3km =========---- 55%     │
│                                      │
│  남은 시간: 3일 12시간                │
│  1위까지 3.6km 남았어요!              │
└──────────────────────────────────────┘
```

#### 2. 새 챌린지 만들기 CTA

| 요소 | 상세 |
|------|------|
| 위치 | 진행 중 챌린지 하단 |
| 형태 | Primary 컬러 버튼 "새 챌린지 만들기" |
| 탭 액션 | 챌린지 생성 바텀시트 |
| 생성 항목 | 챌린지 유형 + 목표 설정 + 기간 + 친구 초대 |
| 친구 초대 | 팔로잉 리스트에서 선택 + 카카오톡 공유 |

#### 챌린지 생성 흐름

```
[새 챌린지 만들기]
  → Step 1: 유형 선택 (거리 대결 / 연속 산책 / 월간 목표)
  → Step 2: 목표 설정 (거리 km / 일수 / 횟수)
  → Step 3: 기간 설정 (1주 / 2주 / 1개월)
  → Step 4: 친구 초대 (팔로잉 목록 / 카카오톡 공유)
  → 생성 완료 → 참가자 알림 발송
```

#### 3. 완료된 챌린지 (Completed Challenges)

| 요소 | 상세 |
|------|------|
| 형태 | 접이식 섹션 "완료된 챌린지 (12개)" |
| 카드 내용 | 챌린지명 + 최종 결과 + 등수 + 뱃지 획득 여부 |
| 결과 표시 | 메달 아이콘 + 최종 기록 |

#### 이 콘텐츠가 있는 이유

> 챌린지 탭은 NRC의 **Challenges 기능을 직접 차용**한 것으로, 소셜 경쟁을 구조화하여 제공한다.
>
> - **사용자 행동 심리학**: NRC에서 친구와 주간 마일리지를 겨루는 Challenge는 가장 인기 있는 기능 중 하나다. Commitment and Consistency(약속과 일관성)에 의해 챌린지에 참여하면 "포기하면 부끄럽다"는 사회적 압력이 산책 이행률을 높인다.
> - **NRC 참조**: NRC의 Challenge는 (1) 친구 초대, (2) 기간 설정, (3) 실시간 진행 비교, (4) 완료 후 결과 요약이라는 4단계로 구성된다. 멍이랑도 동일한 구조를 따르되, "산책 거리" "산책 횟수" "연속 산책일"이라는 반려동물 산책 특화 지표를 사용한다.
> - **비즈니스 목표**: 챌린지는 주간 리텐션의 핵심 드라이버다. 또한 챌린지 초대는 친구에게 앱 설치를 유도하는 자연스러운 바이럴 채널이다.

### 클럽 탭 (Clubs)

#### 1. 내 클럽 (My Clubs)

| 요소 | 상세 |
|------|------|
| 형태 | 카드형 리스트 |
| 카드 내용 | 클럽명 + 대표 이미지 + 멤버 수 + 이번 주 활동량 |
| 탭 액션 | 클럽 상세 화면으로 이동 |
| 최대 가입 수 | 5개 클럽 |
| 역할 | 클럽장 / 운영진 / 멤버 |

#### 클럽 상세 화면

| 요소 | 상세 |
|------|------|
| 클럽 헤더 | 클럽명 + 대표 이미지 + 설명 |
| 멤버 리스트 | 프로필 사진 + 이름 + 이번 주 기록 |
| 클럽 리더보드 | 클럽 내 주간 순위 |
| 클럽 피드 | 멤버들의 산책 기록 (클럽 전용) |
| 클럽 챌린지 | 클럽 단위 챌린지 개설 가능 |
| 정기 모임 | 매주/매월 정기 산책 모임 스케줄 |

#### 2. 추천 클럽 (Recommended Clubs)

| 요소 | 상세 |
|------|------|
| 형태 | 가로 스크롤 카드 리스트 |
| 추천 기준 | 동네 기반 / 견종 기반 / 산책 스타일 기반 |
| 카드 내용 | 클럽명 + 멤버 수 + 설명 + [가입하기] |

#### 이 콘텐츠가 있는 이유

> 클럽 탭은 NRC의 **Club 기능을 반려동물 산책에 적용**한 것이다. Belonging(소속감 욕구)은 Maslow의 욕구 계층 3단계로, 클럽 멤버십은 앱 이탈을 방지하는 강력한 Lock-in 요소다. 기존의 "산책 모임 게시판"을 NRC 스타일의 클럽 시스템으로 리디자인하여, 단발성 모임이 아닌 지속적인 커뮤니티를 형성한다.

### 가이드 산책 탭 (Guided Walks)

#### 1. 추천 산책 프로그램 (Featured Walk Program)

| 요소 | 상세 |
|------|------|
| 형태 | 대형 카드 (가로 풀너비, 높이 200px) |
| 내용 | 프로그램명 + 소개 + 대표 이미지 + 난이도/시간/거리 |
| 탭 액션 | 프로그램 상세 → 산책 시작 |
| 업데이트 | 주 1회 새 프로그램 추천 |

#### 2. 산책 프로그램 그리드 (Walk Programs Grid)

| 요소 | 상세 |
|------|------|
| 형태 | 2열 그리드 카드 |
| 카드 내용 | 썸네일 + 프로그램명 + 난이도 + 예상 시간 + 거리 |
| 필터 | [전체] [초급] [중급] [고급] / [30분] [60분] [90분+] |
| 정렬 | 추천순 / 인기순 / 최신순 |

#### 산책 프로그램 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| 동네 탐험 | 새로운 동네 산책 코스 | "성수동 카페거리 산책" |
| 자연 힐링 | 공원/산/해변 산책 | "서울숲 메타세쿼이아길" |
| 체력 단련 | 장거리/고강도 산책 | "한강 10km 챌린지 코스" |
| 초보 가이드 | 산책 입문자용 | "첫 산책 30분 코스" |
| 계절 특화 | 시즌별 특별 코스 | "벚꽃길 봄 산책" |

#### 가이드 산책 진행 방식

```
[프로그램 선택] → [코스 상세 확인]
  → 코스 지도 + 경유 포인트 + 예상 시간 + 난이도
  → [산책 시작하기]
  → 산책 트래킹 (Tab 3) 진입 + 가이드 경로 오버레이
  → 경유 포인트 도달 시 진동 알림
  → 산책 완료 → "가이드 산책 완료" 특별 카드 생성
```

#### 이 콘텐츠가 있는 이유

> 가이드 산책 탭은 NRC의 **Guided Runs 기능을 반려동물 산책에 적용**한 것이다. NRC Guided Runs는 코치의 음성 가이드와 함께 다양한 러닝 프로그램을 제공하여, "무엇을 해야 할지 모르겠다"는 Decision Fatigue(결정 피로)를 해소한다. 멍이랑은 음성 가이드 대신 경로 오버레이와 경유 포인트 알림으로 가이드를 제공하고, 지역 기반 큐레이션으로 차별화한다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
[1순위 - 일상적 소비 (피드)]
  └── 주간 리더보드 (경쟁 동기)
  └── 라이브 산책 인디케이터 (실시간 FOMO)
  └── 활동 피드 (콘텐츠 소비 + 리액션)

[2순위 - 목표 지향 (챌린지)]
  └── 진행 중 챌린지 (단기 목표)
  └── 새 챌린지 만들기 (사회적 약속)

[3순위 - 소속감 (클럽)]
  └── 내 클럽 (정체성)
  └── 추천 클럽 (네트워크 확장)

[4순위 - 탐색 (가이드 산책)]
  └── 추천 프로그램 (Decision Fatigue 해소)
  └── 프로그램 그리드 (새로운 경험)
```

핵심 원칙: **피드(일상) → 챌린지(경쟁) → 클럽(소속) → 가이드(탐색) 순서**로 서브 탭을 배치한다.

### 3.2 인터랙션 패턴

| 패턴 | 적용 위치 | 설명 |
|------|----------|------|
| Swipe Tab Navigation | 서브 탭 | 좌우 스와이프로 탭 전환 |
| Pull-to-Refresh | 피드 전체 | 새 피드 콘텐츠 로드 |
| Infinite Scroll | 피드 리스트 | 하단 도달 시 다음 페이지 자동 로드 |
| Emoji Reaction | 피드 카드 | 이모지 탭으로 빠른 리액션 |
| Cheer Button | 피드 카드 | 응원 보내기 (푸시 알림 연동) |
| Progress Bar | 챌린지 카드 | 실시간 진행률 시각화 |
| Bottom Sheet | 챌린지 생성, 리더보드 상세 | 바텀시트로 추가 UI 표시 |
| Haptic Feedback | 응원, 리액션 탭 | 인터랙션 확인 진동 |

### 3.3 접근성 고려사항

| 항목 | 대응 |
|------|------|
| 색상 대비 | 이모지 리액션 카운터, 진행률 바 텍스트 4.5:1 이상 |
| 터치 영역 | 이모지 리액션, 응원 버튼 최소 44x44px |
| 스크린 리더 | "초코의 산책, 3.2킬로미터, 45분, 응원 12개" |
| 이미지 대체 텍스트 | 경로 맵에 alt 제공 |
| 모션 감소 | 축하 애니메이션을 정적 상태 변경으로 대체 |

## 5. 안전 및 프라이버시

### 5.1 프라이버시 원칙

| 원칙 | 상세 |
|------|------|
| 반려동물 중심 프로필 | 견주 개인정보(실명, 사진, 성별, 나이) 비공개 |
| 위치 정보 보호 | 정확한 출발지/도착지 좌표 비공개 (200m 범위 퍼징) |
| 라이브 위치 동의 | 실시간 산책 위치 공유는 명시적 동의 후에만 활성화 |
| 차단 기능 | 상호 피드/추천에서 완전 제외, 차단 사실 비공개 |

### 5.2 신고 시스템

| 유형 | 처리 |
|------|------|
| 부적절한 콘텐츠 | 24시간 내 검토 |
| 스팸/광고 | 자동 필터 + 수동 검토 |
| 괴롭힘 | 즉시 숨김 + 검토 |
| 동물 학대 의심 | 긴급 검토 + 관계기관 연계 |
