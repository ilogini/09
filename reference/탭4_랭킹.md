# 랭킹 화면 (Tab 4) - 멍이랑 (withbowwow)

> 산책의 게이미피케이션 핵심. 경쟁과 동기부여의 중심.
> "이번 주 동네 1위"를 향한 도전이 매일의 산책을 이끈다.

---

## 1. 페이지 개요

### 1.1 목적

랭킹 화면은 멍이랑 앱의 **게이미피케이션 엔진**이다. 사용자 간 건강한 경쟁을 촉진하여 산책 빈도와 거리를 높이는 것이 목적이다. "내가 동네에서 몇 등인지" 확인하는 행위 자체가 앱 재방문의 강력한 이유가 되며, "1등을 따라잡겠다" 또는 "1등을 지키겠다"는 목표가 매일의 산책을 이끈다. Nike Run Club의 Leaderboard와 Strava의 Segment Competition을 반려동물 산책에 특화하여 설계한다.

### 1.2 핵심 사용자 니즈

| 니즈 | 설명 | 우선순위 |
|------|------|---------|
| 내 순위 확인 | 동네/전체에서 내 위치 즉시 확인 | 최상위 |
| 경쟁 대상 발견 | 나보다 약간 위에 있는 사용자 확인 | 상위 |
| 성장 확인 | 지난 주 대비 순위 변동 확인 | 상위 |
| 지역 커뮤니티 | 같은 동네 산책 사용자 전체 파악 | 중위 |
| 명예 획득 | 장기간 상위 랭커로서의 인정 | 중위 |

### 1.3 진입 경로

| 경로 | 빈도 | 컨텍스트 |
|------|------|---------|
| 하단 탭 바 "랭킹" 탭 | 높음 | 일상적 순위 확인 |
| 홈 → 랭킹 미리보기 → "전체보기" | 높음 | 홈에서 순위 확인 후 상세 진입 |
| 푸시 알림 (랭킹 변동) | 중간 | "순위가 올랐어요!" 딥링크 |
| 산책 완료 → 랭킹 업데이트 | 중간 | 산책 후 변경된 순위 확인 |

## 2. 화면 구성

### 섹션 1: 상단 네비게이션 + 기간 선택 (Header + Period Selector)

#### 콘텐츠 구성

| 요소 | 위치 | 상세 |
|------|------|------|
| 페이지 타이틀 | 좌측 | "랭킹" (Bold, 24pt) |
| 기간 표시 | 타이틀 하단 | "2월 2주차 (2/10~2/16)" |
| 세그먼트 컨트롤 | 헤더 하단 | [주간] / [월간] / [전체] |

#### 세그먼트 컨트롤 상세

| 기간 | 집계 방식 | 리셋 시점 | 주 사용 패턴 |
|------|----------|----------|------------|
| 주간 | 월요일 00:00 ~ 일요일 23:59 | 매주 월요일 | 가장 활발한 경쟁 (추천 기본값) |
| 월간 | 매월 1일 ~ 말일 | 매월 1일 | 중기 목표 |
| 전체 | 가입 이후 누적 | 리셋 없음 | 장기 충성 사용자 확인 |

#### 이 콘텐츠가 있는 이유

> 세그먼트 컨트롤은 사용자의 **경쟁 프레임**을 선택하게 한다.
>
> - **사용자 행동 심리학**: Fresh Start Effect(새 출발 효과)에 따르면 사람들은 새로운 시간 단위(새 주, 새 달)가 시작되면 동기가 갱신된다. 주간 랭킹은 매주 리셋되므로 "지난 주에 못 했어도 이번 주에 다시 도전"이 가능하다. 전체 랭킹은 장기 충성 사용자에게 "1년간의 노력"이 인정받는 공간을 제공한다.
> - **비즈니스 목표**: 주간 랭킹은 주 단위 리텐션을 직접 드라이브한다. "이번 주 금요일인데 아직 3위, 주말에 10km만 더 걸으면 1위 가능"이라는 계산이 주말 산책을 유도한다. 월간/전체 랭킹은 장기 사용자의 이탈을 방지한다.
> - **벤치마킹**: Nike Run Club은 주간/월간 Leaderboard를 제공한다. Strava는 "This Week" / "This Year" / "All Time"으로 기간을 구분한다. Apple Fitness의 Competition은 7일 단위 경쟁을 기본으로 한다.
> - **기본값이 "주간"인 이유**: 주간이 가장 활발한 경쟁 단위이며, Fresh Start Effect가 가장 자주 작동한다. 월간은 기간이 길어 초반에 격차가 벌어지면 동기가 감소할 수 있다.

### 섹션 2: 내 순위 하이라이트 카드 (My Rank Highlight)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 카드 형태 | 고정 카드 (스크롤해도 상단에 고정, Sticky Header) |
| 배경 | Primary 컬러 그라데이션 |
| 내 순위 | 대형 숫자 "#7" (48pt) |
| 순위 변동 | "↑ 3" (지난 기간 대비 변동, 초록/빨강) |
| 내 반려동물 프로필 | 원형 사진 + 이름 |
| 내 기록 | "이번 주 18.5km · 5회 산책" |
| 목표까지 거리 | "6위까지 2.3km 남았어요!" |
| 동기부여 메시지 | "조금만 더 걸으면 TOP 5 진입!" |

#### 카드 레이아웃

```
┌──────────────────────────────────────┐
│  ┌──────┐                            │
│  │[🐕📷]│  #7  ↑3                    │  ← Primary 배경
│  │ 초코  │  이번 주 18.5km · 5회       │
│  └──────┘  🎯 6위까지 2.3km!          │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 내 순위 카드는 랭킹 화면의 **앵커(anchor)**이자 **가장 강력한 동기부여 요소**다.
>
> - **사용자 행동 심리학**: Self-Reference Effect(자기참조 효과)에 따르면 자신과 관련된 정보를 가장 먼저 그리고 가장 오래 처리한다. 다른 사람의 순위보다 "내 순위"가 가장 중요하므로 최상단에 고정 배치한다. "6위까지 2.3km"라는 구체적인 Gap 정보는 Goal Gradient Effect를 극대화한다. 목표가 구체적이고 달성 가능해 보일 때 행동 의지가 가장 강해진다.
> - **비즈니스 목표**: 내 순위 카드가 산책 행동을 직접 유도한다. "2.3km만 더 걸으면 6위"라는 메시지는 "오늘 한 번 더 산책하자"는 결심으로 이어진다. 이는 일일 산책 횟수 증가 → 앱 사용 시간 증가 → 광고 노출 증가의 선순환을 만든다.
> - **벤치마킹**: Apple Fitness의 Competition에서는 "You're behind by 143 points"처럼 구체적인 차이를 보여준다. 이 정보가 "오늘 운동 한 번 더 해야겠다"를 결정짓는다. Nike Run Club도 친구와의 주간 마일리지 차이를 수치로 보여준다.
> - **순위 변동 표시**: "↑3"은 Positive Reinforcement(긍정적 강화)로, 순위가 올랐을 때 성취감을 제공한다. "↓2"도 Loss Aversion(손실 회피)을 활용하여 "떨어진 순위를 복구해야 한다"는 동기를 만든다.

### 섹션 3: 랭킹 리스트 (Ranking List)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 리스트 형태 | 수직 스크롤 리스트 |
| 항목 구성 | 순위 + 프로필 사진 + 닉네임(반려동물) + 거리 + 산책 횟수 |
| TOP 3 강조 | 금/은/동 메달 아이콘 + 배경 색상 차별화 |
| 내 순위 강조 | Primary 컬러 배경 + Bold 텍스트 |
| 페이지네이션 | 20명씩 로드 + 무한 스크롤 |
| 빈 상태 | "아직 이번 주 산책 기록이 없어요" |

#### TOP 3 + 일반 랭킹 레이아웃

```
┌──────────────────────────────────────┐
│  🥇 1  [🐕] 뽀삐아빠     52.3km  12회 │  ← 골드 배경
│  🥈 2  [🐶] 초코맘       48.1km  10회 │  ← 실버 배경
│  🥉 3  [🐩] 달이아빠     45.7km   9회 │  ← 브론즈 배경
├──────────────────────────────────────┤
│    4  [🐕] 몽이맘       42.1km   8회 │
│    5  [🐶] 콩이아빠     39.8km   9회 │
│    6  [🐩] 두부맘       20.8km   6회 │
│ ★ 7  [🐕] 나 (초코)    18.5km   5회 │  ← Primary 하이라이트
│    8  [🐶] 별이맘       17.2km   5회 │
│   ...                                │
└──────────────────────────────────────┘
```

#### 랭킹 항목 탭 액션

```
[사용자 항목 탭]
  → 바텀시트: 반려동물 프로필 요약
    - 반려동물 사진, 이름, 견종
    - 이번 기간 기록 (거리, 횟수)
    - [팔로우] 버튼
    - [프로필 보기] 버튼
```

#### 이 콘텐츠가 있는 이유

> 랭킹 리스트는 **사회 비교(Social Comparison)**의 직접적 구현체다.
>
> - **사용자 행동 심리학**: Festinger의 Social Comparison Theory에서 가장 강력한 동기부여는 "나와 비슷한 수준의 사람과 비교"할 때 발생한다. 전체 리스트에서 내 순위 위아래에 있는 사용자들이 가장 강력한 경쟁 대상이 된다. TOP 3의 금/은/동 시각적 강조는 Aspirational Target(열망 대상)을 만들어 "나도 저기까지 올라가고 싶다"는 장기 목표를 설정하게 한다.
> - **비즈니스 목표**: 랭킹 리스트에서 다른 사용자 프로필을 탭하면 소셜 연결(팔로우)로 이어진다. 이는 소셜 그래프 확장 → 피드 활성화 → 앱 체류 시간 증가의 선순환을 만든다. 또한 상위 랭커에게 "프리미엄 뱃지" 등 특별 보상을 제공하여 경쟁 욕구를 자극하고, 이를 통해 프리미엄 전환을 유도할 수 있다.
> - **벤치마킹**: Strava의 Segment Leaderboard는 특정 구간의 최고 기록을 보여주며, 각 항목을 탭하면 해당 사용자의 활동 상세를 볼 수 있다. Nike Run Club의 Leaderboard는 친구 중심이지만, 멍이랑은 "동네" 중심으로 확장하여 실제 이웃과의 경쟁감을 형성한다.
> - **반려동물 프로필 표시**: 사람이 아닌 반려동물끼리의 경쟁으로 프레이밍함으로써, 공격적 경쟁이 아닌 유쾌한 경쟁 분위기를 조성한다. "우리 초코가 뽀삐한테 졌네!" 같은 대화가 자연스럽다.

### 섹션 4: 동네별 랭킹 필터 (Region Filter)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 필터 위치 | 리스트 상단 칩(Chip) 형태 |
| 기본값 | "내 동네" (GPS 기반 자동 설정) |
| 필터 옵션 | 내 동네 (동 단위) / 내 구 (구 단위) / 전국 |
| 지역 표시 | "성동구 성수동" 등 행정구역명 |
| 지역 변경 | 칩 탭 → 지역 선택 바텀시트 |

#### 지역 범위별 사용자 수 예시

| 범위 | 예시 | 예상 참가자 수 (초기) |
|------|------|---------------------|
| 동 단위 | 성수동1가 | 10~50명 |
| 구 단위 | 성동구 | 50~300명 |
| 전국 | 대한민국 전체 | 1,000~10,000명 |

#### 이 콘텐츠가 있는 이유

> 지역 필터는 **경쟁의 밀도**를 조절하는 도구다.
>
> - **사용자 행동 심리학**: Optimal Challenge(최적 도전) 이론에 따르면 너무 쉽거나 너무 어려운 경쟁은 동기를 감소시킨다. 전국 랭킹에서 1000등이면 동기가 없지만, 동네 랭킹에서 7등이면 "5등까지 올라가자"는 도전이 생긴다. 사용자가 자신에게 맞는 경쟁 범위를 선택할 수 있도록 하여, 모든 수준의 사용자에게 적절한 도전을 제공한다.
> - **비즈니스 목표**: 동네 단위 경쟁은 지역 커뮤니티 형성을 촉진한다. "성수동 1등"이라는 로컬 명성은 전국 1등보다 오히려 더 체감 가능한 보상이다. 이는 소셜 탭의 "산책 모임"으로 연결되어 오프라인 만남까지 이어질 수 있다.
> - **벤치마킹**: 당근마켓이 "동네" 단위의 서비스로 성공한 것처럼, 지역 기반 경쟁은 한국 사용자에게 특히 효과적이다. Strava의 "Local Legends" 기능도 특정 구간에서 가장 많이 달린 사람을 지역 전설로 인정한다.

### 섹션 5: 명예의 전당 배너 (Hall of Fame Banner)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 배너 형태 | 가로 풀 너비 카드, 골드 그라데이션 배경 |
| 헤드라인 | "명예의 전당" |
| 서브텍스트 | "역대 최고의 산책러들을 만나보세요" |
| 탭 액션 | 명예의 전당 별도 스크린으로 이동 |

#### 명예의 전당 스크린 (별도 화면)

| 카테고리 | 설명 | 업데이트 |
|---------|------|---------|
| 이번 달 MVP | 월간 최다 거리 1위 | 매월 1일 갱신 |
| 최장 연속 산책 | 역대 최장 연속일 기록 | 실시간 |
| 최장 거리 1회 산책 | 1회 산책 최장 거리 | 실시간 |
| 역대 누적 거리 TOP 10 | 가입 이후 누적 거리 | 실시간 |
| 시즌 챔피언 | 분기별 랭킹 1위 | 분기별 갱신 |

#### 명예의 전당 카드 레이아웃

```
┌──────────────────────────────────────┐
│  👑 이번 달 MVP                       │
│  ┌──────┐                            │
│  │[🐕📷]│  뽀삐아빠                    │
│  │ 뽀삐  │  2월: 213.7km · 28회 산책   │
│  └──────┘  "비가 와도 뽀삐랑 산책!"    │
└──────────────────────────────────────┘
```

#### 이 콘텐츠가 있는 이유

> 명예의 전당은 **장기적 목표(aspiration)**를 제공하는 영역이다.
>
> - **사용자 행동 심리학**: Self-Determination Theory(자기결정 이론)의 3가지 기본 욕구 중 "유능감(Competence)"과 "관계성(Relatedness)"을 동시에 충족한다. 명예의 전당에 이름을 올린다는 것은 커뮤니티 내에서의 인정(Recognition)이며, 이는 외적 보상(포인트, 뱃지)보다 더 강력한 내재적 동기를 만든다. 또한 "이번 달 MVP"는 매월 갱신되므로 누구나 도전 가능하다는 인식을 심어준다.
> - **비즈니스 목표**: 명예의 전당 등재자는 앱의 "파워 유저"로서 커뮤니티의 핵심 역할을 한다. 이들은 가장 높은 DAU, 가장 많은 콘텐츠 생산, 가장 높은 프리미엄 전환율을 보이는 세그먼트다. 명예의 전당은 이런 파워 유저를 인정하고 유지하는 메커니즘이다.
> - **벤치마킹**: Strava의 "Local Legend" 칭호는 특정 세그먼트에서 가장 많이 달린 사람에게 부여된다. 이 칭호가 "그 구간을 더 달리고 싶다"는 동기를 만든다. Apple Fitness의 "Longest Move Streak" 기록도 명예의 전당 성격을 가진다.

### 섹션 6: 랭킹 기준 정보 (Ranking Criteria Info)

#### 콘텐츠 구성

| 요소 | 상세 |
|------|------|
| 위치 | 리스트 하단 (풋터 영역) |
| 형태 | 접이식 (Collapsible) 정보 패널 |
| 내용 | 랭킹 기준 설명, 집계 방법, 부정행위 방지 안내 |

#### 랭킹 기준 상세

| 기준 | 설명 |
|------|------|
| 주 지표 | 산책 거리 (km) - 기본 정렬 기준 |
| 보조 지표 | 산책 횟수 (동일 거리 시 횟수가 많은 쪽 우선) |
| 최소 기준 | 주간 1회 이상 산책해야 랭킹 표시 |
| 부정행위 필터 | 시속 15km 이상 구간 제외, GPS 스푸핑 감지 |
| 집계 시점 | 실시간 (산책 완료 즉시 반영) |

#### 이 콘텐츠가 있는 이유

> 랭킹 기준의 투명성은 사용자 **신뢰**의 기반이다.
>
> - **공정성 인식**: 랭킹 시스템이 공정하다고 느껴야 경쟁에 참여한다. 기준이 불분명하거나 부정행위가 의심되면 사용자는 경쟁을 포기한다. 투명한 기준 공개와 부정행위 방지 안내가 신뢰를 구축한다.
> - **비즈니스 목표**: 부정행위(차량 이동을 산책으로 기록 등)가 횡행하면 정직한 사용자가 이탈한다. 시속 15km 이상 필터링, GPS 스푸핑 감지 등 기술적 대응을 사전에 공지하여 공정한 경쟁 환경을 보장한다.

## 3. UX 설계 의도

### 3.1 정보 계층 구조

```
[1순위 - 나의 상태]
  └── 기간 선택 (주간/월간/전체)

  └── 랭킹 리스트 (TOP 3 강조 + 전체)

  └── 명예의 전당 배너

[4순위 - 부가 정보]
  └── 랭킹 기준 안내
```

### 랭킹 집계 SQL 예시

```sql
-- 주간 랭킹 집계
SELECT
  u.id,
  u.nickname,
  p.name AS pet_name,
  p.photo_url,
  SUM(w.distance_m) / 1000.0 AS total_km,
  COUNT(w.id) AS walk_count,
  RANK() OVER (ORDER BY SUM(w.distance_m) DESC) AS rank
FROM walks w
JOIN users u ON w.user_id = u.id
JOIN pets p ON w.pet_id = p.id
WHERE w.started_at >= date_trunc('week', CURRENT_DATE)
  AND w.distance_m >= 500  -- 최소 500m 필터
  AND u.region = '성동구'  -- 지역 필터
GROUP BY u.id, u.nickname, p.name, p.photo_url
ORDER BY total_km DESC;
```
